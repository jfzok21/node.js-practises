# 第 2 週今日練習題：PostgreSQL 基本 CRUD 操作

## 題目說明

建立一個簡單的「書籍管理 API」，使用 Express.js 作為後端框架，並透過 PostgreSQL 儲存書籍資料。練習從資料庫連線到完整的 CRUD 操作，讓你熟悉 Node.js 操作關聯式資料庫的基本流程。

---

## 技術要求

- **Express.js**：建立 RESTful API 路由，處理 HTTP 請求與回應
- **PostgreSQL**：使用 `pg`（node-postgres）套件連線，執行 SQL 語句進行資料讀寫
- **資料庫**：建立 `books` 資料表，包含書名、作者、出版年份等欄位

---

## 具體任務

1. **資料庫準備**
   - 安裝並啟動 PostgreSQL，建立資料庫 `book_db`
   - 建立資料表：
     ```sql
     CREATE TABLE books (
       id SERIAL PRIMARY KEY,
       title VARCHAR(255) NOT NULL,
       author VARCHAR(255) NOT NULL,
       year INTEGER,
       created_at TIMESTAMP DEFAULT NOW()
     );
     ```

2. **建立 Express 專案與資料庫連線**
   - 初始化 Node.js 專案，安裝 `express`、`pg`、`dotenv`
   - 使用 `.env` 管理資料庫連線參數（HOST、PORT、USER、PASSWORD、DATABASE）
   - 建立 `db.js`，匯出 PostgreSQL Pool 連線物件

3. **實作 CRUD API**

   | 方法     | 路徑           | 說明             |
   |----------|----------------|------------------|
   | `GET`    | `/books`       | 取得所有書籍列表  |
   | `GET`    | `/books/:id`   | 取得單本書籍詳細  |
   | `POST`   | `/books`       | 新增一本書籍      |
   | `PUT`    | `/books/:id`   | 更新書籍資料      |
   | `DELETE` | `/books/:id`   | 刪除一本書籍      |

4. **回應格式**
   - 成功時回傳 JSON，例如：
     ```json
     { "success": true, "data": { "id": 1, "title": "..." } }
     ```
   - 找不到資源時回傳 404：
     ```json
     { "success": false, "message": "Book not found" }
     ```

5. **測試 API**
   - 使用 `curl` 或 Postman 測試每個端點，確認 INSERT / SELECT / UPDATE / DELETE 均正常運作

---

## 提示

- 使用 `pool.query(sql, [params])` 執行參數化查詢，避免 SQL Injection
- `SERIAL` 型別會自動遞增，INSERT 後可用 `RETURNING *` 取回新增的完整資料列
- `pg.Pool` 適合 API 伺服器使用（自動管理連線數量），比 `pg.Client` 更推薦
- 記得在 `.gitignore` 加入 `.env`，避免資料庫密碼上傳到 GitHub
- 可用 `process.env.PORT || 3000` 讓 port 從環境變數讀取

---

## 進階挑戰（選做）

- 新增資料驗證：`title` 與 `author` 不可為空，`year` 必須是合理的年份（如 1000～2030），驗證失敗時回傳 400 與錯誤訊息
- 支援部分更新（PATCH），只更新請求體中提供的欄位
- 新增 `GET /books?author=王小明` 查詢功能，依作者名稱篩選結果（使用 `WHERE author ILIKE $1`）
- 將資料庫操作抽離成獨立的 Repository 層（`booksRepository.js`），讓路由處理器更簡潔
