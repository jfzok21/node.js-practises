# 第 1 週今日練習題：Express 基礎路由 — 書籍清單 API

## 題目說明

使用 Express.js 建立一個「書籍清單（Books）」REST API，以純記憶體陣列儲存資料（不需接資料庫）。實作 GET、POST 兩種 HTTP 方法，熟悉路由定義、JSON 請求解析與回應格式的設計。

---

## 技術要求

- **Express.js**：建立 HTTP 伺服器、定義路由（`app.get`、`app.post`）、解析 JSON 請求體（`express.json()`）
- **PostgreSQL**：本週不使用，資料以 JS 陣列暫存即可

---

## 具體任務

1. **初始化專案**
   - 在 `week1-books/` 資料夾下執行 `npm init -y`，安裝 `express`
   - 建立 `index.js` 作為主程式進入點

2. **啟動 Express 伺服器**
   - 監聽埠號 `3000`
   - 伺服器啟動後印出 `Books API running on http://localhost:3000`

3. **GET `/books`**
   - 回傳所有書籍清單，格式：
     ```json
     {
       "books": [
         { "id": 1, "title": "JavaScript 入門", "author": "小明", "year": 2023 }
       ]
     }
     ```
   - 若清單為空，回傳 `{ "books": [] }`

4. **POST `/books`**
   - 接收 JSON 請求體 `{ "title": "...", "author": "...", "year": 2024 }`
   - `id` 自動遞增（從 1 開始）
   - 新增成功後以 **HTTP 201** 回傳新建立的物件：
     ```json
     { "id": 2, "title": "Node.js 實戰", "author": "小華", "year": 2024 }
     ```
   - 若 `title` 或 `author` 任一未提供，回傳 **HTTP 400**：
     ```json
     { "error": "title 與 author 為必填欄位" }
     ```

5. **GET `/books/:id`**
   - 依 `id` 取得單筆書籍資料
   - 找不到時回傳 **HTTP 404**：
     ```json
     { "error": "找不到 id 為 5 的書籍" }
     ```

6. **GET `/books/search?author=小明`**
   - 以查詢字串 `?author=` 過濾書籍（部分比對即可）
   - 回傳符合條件的陣列，格式同 `GET /books`

---

## 提示

- 在所有路由之前加上 `app.use(express.json())` 才能讀取 `req.body`
- `req.params.id` 是字串，與數字 `id` 比較前請用 `Number(req.params.id)` 轉型
- 查詢字串可用 `req.query.author` 取得，記得處理未傳入的情況（回傳全部）
- 注意路由順序：`/books/search` 必須定義在 `/books/:id` **之前**，否則 `search` 會被當成 id

---

## 進階挑戰（選做）

- **PUT `/books/:id`**：接收完整書籍物件，覆蓋指定 id 的書籍，找不到時回傳 404
- **DELETE `/books/:id`**：刪除指定書籍，成功回傳 **HTTP 204**（No Content）
- 試著用 `curl` 或 Postman 測試所有路由，驗證各種邊界情況（空白 title、不存在的 id 等）
