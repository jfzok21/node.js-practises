# 第 1 週今日練習題：打造你的第一個 Express API

## 題目說明

建立一個簡單的「書單管理 API」，練習 Express 的基礎路由設定，並透過 GET / POST 等 HTTP 方法回傳 JSON 資料。所有資料暫存在記憶體（陣列）中，不需要連接資料庫。

---

## 技術要求

- **Express.js**：設定 Express 應用程式、定義路由、回傳 JSON 回應
- **PostgreSQL**：本週不需要，專注在 Express 路由基礎

---

## 具體任務

1. **初始化專案**
   - 建立新資料夾 `book-api`，執行 `npm init -y`
   - 安裝 `express`（`npm install express`）
   - 建立 `index.js` 作為程式進入點

2. **建立基本伺服器**
   - 使用 `express()` 建立 app 實例
   - 監聽 `PORT 3000`，啟動後在終端機印出 `伺服器已啟動：http://localhost:3000`

3. **實作 GET 路由**
   - `GET /` → 回傳 `{ "message": "歡迎使用書單 API" }`
   - `GET /books` → 回傳所有書籍陣列（初始可放 2～3 筆假資料）
   - `GET /books/:id` → 根據 id 回傳單一書籍；若找不到，回傳 `404` 與 `{ "error": "找不到該書籍" }`

4. **實作 POST 路由**
   - `POST /books` → 接收 request body（需包含 `title`、`author`），新增到陣列中並回傳 `201` 狀態碼與新增的書籍物件
   - 若缺少 `title` 或 `author`，回傳 `400` 與 `{ "error": "title 與 author 為必填欄位" }`

5. **測試 API**
   - 使用 `curl` 或 Postman 測試所有路由
   - 確認每個路由都回傳正確的 HTTP 狀態碼與 JSON 格式

---

## 提示

- 記得加上 `express.json()` middleware 才能解析 POST 的 JSON body：
  ```js
  app.use(express.json())
  ```
- 書籍資料結構建議：
  ```js
  { id: 1, title: "Node.js 實戰", author: "John" }
  ```
- 用 `res.status(404).json(...)` 同時設定狀態碼與回傳 JSON
- `req.params.id` 取得的值是**字串**，比較時記得轉型：`Number(req.params.id)`

---

## 進階挑戰（選做）

- 實作 `DELETE /books/:id`：刪除指定書籍，若不存在回傳 404
- 實作 `PUT /books/:id`：更新書籍的 title 或 author
- 為每本新增的書籍自動產生遞增的 `id`（不要手動指定）
- 新增 `GET /books?author=John` 的查詢字串功能，篩選特定作者的書籍
