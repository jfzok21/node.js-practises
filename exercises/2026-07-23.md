# 第 3 週今日練習題：Express 路由參數與查詢字串

## 題目說明

建立一個管理「電影清單」的 Express API，深入練習路由參數（`:id`）與查詢字串（`?search=`、`?genre=`、`?year=`）的使用。透過本練習，你將學會如何彈性地接收使用者傳入的參數，並根據不同條件回傳對應的資料。

---

## 技術要求

- **Express.js**：使用路由參數（Route Parameters）與查詢字串（Query String）處理動態請求
- **PostgreSQL**：使用 `pg` 套件，依據參數動態組合 SQL 查詢條件

---

## 具體任務

1. **建立資料庫與資料表**
   - 在 PostgreSQL 中建立 `movies` 資料表：
     ```sql
     CREATE TABLE movies (
       id SERIAL PRIMARY KEY,
       title VARCHAR(255) NOT NULL,
       genre VARCHAR(100),
       director VARCHAR(255),
       release_year INTEGER,
       rating NUMERIC(3, 1),
       created_at TIMESTAMP DEFAULT NOW()
     );
     ```
   - 插入至少 5 筆測試資料：
     ```sql
     INSERT INTO movies (title, genre, director, release_year, rating) VALUES
       ('Full Metal Jacket', 'war', 'Stanley Kubrick', 1987, 8.3),
       ('The Dark Knight', 'action', 'Christopher Nolan', 2008, 9.0),
       ('Parasite', 'thriller', 'Bong Joon-ho', 2019, 8.5),
       ('Spirited Away', 'animation', 'Hayao Miyazaki', 2001, 8.6),
       ('Interstellar', 'sci-fi', 'Christopher Nolan', 2014, 8.6);
     ```

2. **實作以下 API 路由**

   | 方法 | 路徑              | 功能                               |
   |------|-------------------|------------------------------------|
   | GET  | `/movies`         | 取得所有電影（支援查詢字串篩選）   |
   | GET  | `/movies/:id`     | 依 ID 取得單一電影                 |
   | GET  | `/directors/:name/movies` | 取得特定導演的所有電影   |

3. **`GET /movies` 需支援以下查詢字串**

   | 查詢參數  | 說明                             | 範例                         |
   |-----------|----------------------------------|------------------------------|
   | `search`  | 以關鍵字模糊搜尋電影標題         | `/movies?search=dark`        |
   | `genre`   | 依類型篩選                       | `/movies?genre=action`       |
   | `year`    | 依上映年份篩選                   | `/movies?year=2019`          |
   | 多重組合  | 同時套用多個條件                 | `/movies?genre=sci-fi&year=2014` |

4. **`GET /movies/:id` 的回應格式範例**
   ```json
   {
     "id": 2,
     "title": "The Dark Knight",
     "genre": "action",
     "director": "Christopher Nolan",
     "release_year": 2008,
     "rating": "9.0"
   }
   ```

5. **`GET /directors/:name/movies` 的回應格式範例**
   ```json
   {
     "director": "Christopher Nolan",
     "count": 2,
     "movies": [
       { "id": 2, "title": "The Dark Knight", "release_year": 2008 },
       { "id": 5, "title": "Interstellar", "release_year": 2014 }
     ]
   }
   ```

---

## 提示

- 使用 `req.params.id` 取得路由參數，使用 `req.query.search` 取得查詢字串
- 動態組合 SQL 查詢條件時，推薦做法：
  ```js
  const conditions = [];
  const values = [];

  if (req.query.search) {
    values.push(`%${req.query.search}%`);
    conditions.push(`title ILIKE $${values.length}`);
  }
  if (req.query.genre) {
    values.push(req.query.genre);
    conditions.push(`genre = $${values.length}`);
  }
  if (req.query.year) {
    values.push(parseInt(req.query.year));
    conditions.push(`release_year = $${values.length}`);
  }

  const where = conditions.length > 0 ? `WHERE ${conditions.join(' AND ')}` : '';
  const result = await pool.query(`SELECT * FROM movies ${where}`, values);
  ```
- `ILIKE` 為 PostgreSQL 不分大小寫的模糊比對，等同於 `LIKE` 加上忽略大小寫
- `parseInt()` 可將字串型態的查詢參數轉為整數，避免型態不符的 SQL 錯誤
- 路由順序很重要：`/movies/popular`（靜態路由）若放在 `/movies/:id` 之後，`:id` 會先攔截到 `"popular"` 字串

## 進階挑戰（選做）

- 在 `GET /movies` 加入 `?sort=rating` 與 `?order=desc` 支援，讓使用者可以依欄位排序（注意防止 SQL Injection，僅允許白名單欄位名稱）
- 新增 `GET /movies/genres` 路由，回傳資料庫中所有不重複的電影類型清單
- 當 `?year=` 傳入非數字時，回傳 `400 Bad Request` 並附上清楚的錯誤訊息
- 將查詢邏輯抽出成獨立的 `services/movieService.js` 模組，讓路由更簡潔
