# 第 2 週今日練習題：PostgreSQL 基本 CRUD 操作

## 題目說明

建立一個管理「書籍清單」的 Express API，使用 PostgreSQL 作為資料庫儲存書籍資料。透過本練習，你將學會如何在 Node.js 中連線 PostgreSQL、並實作完整的 CRUD 操作（新增、查詢、更新、刪除）。

---

## 技術要求

- **Express.js**：建立 RESTful API 路由，回傳 JSON 格式資料
- **PostgreSQL**：使用 `pg`（node-postgres）套件連線資料庫，操作 `books` 資料表

---

## 具體任務

1. **建立資料庫與資料表**
   - 在 PostgreSQL 中建立一個名為 `library` 的資料庫
   - 在其中建立 `books` 資料表，欄位如下：
     ```sql
     CREATE TABLE books (
       id SERIAL PRIMARY KEY,
       title VARCHAR(255) NOT NULL,
       author VARCHAR(255) NOT NULL,
       published_year INTEGER,
       created_at TIMESTAMP DEFAULT NOW()
     );
     ```

2. **建立 Express 伺服器並連線資料庫**
   - 安裝套件：`npm install express pg`
   - 使用 `pg.Pool` 建立資料庫連線池
   - 伺服器監聽 `PORT 3000`

3. **實作以下 API 路由**

   | 方法   | 路徑             | 功能             |
   |--------|------------------|------------------|
   | GET    | `/books`         | 取得所有書籍     |
   | GET    | `/books/:id`     | 取得單一書籍     |
   | POST   | `/books`         | 新增一本書籍     |
   | PUT    | `/books/:id`     | 更新書籍資料     |
   | DELETE | `/books/:id`     | 刪除一本書籍     |

4. **POST `/books` 的請求格式範例**
   ```json
   {
     "title": "深入淺出 Node.js",
     "author": "朴靈",
     "published_year": 2013
   }
   ```

5. **確保每個路由都有適當的錯誤處理**
   - 找不到資料時回傳 `404`
   - 伺服器錯誤時回傳 `500`

---

## 提示

- 使用 `pg.Pool` 而非 `pg.Client`，可自動管理連線，適合 API 服務場景
- 在路由中使用參數化查詢（Parameterized Query）防止 SQL Injection：
  ```js
  const result = await pool.query('SELECT * FROM books WHERE id = $1', [id]);
  ```
- 若查詢結果為空（`result.rows.length === 0`），記得回傳 `404`
- 可將 `pool` 放在獨立的 `db.js` 檔案中，方便各路由引用
- 建議的資料庫連線設定方式（使用環境變數）：
  ```js
  const pool = new Pool({
    host: process.env.DB_HOST || 'localhost',
    port: process.env.DB_PORT || 5432,
    database: process.env.DB_NAME || 'library',
    user: process.env.DB_USER || 'postgres',
    password: process.env.DB_PASSWORD || '',
  });
  ```
