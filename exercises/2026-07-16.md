# 第 2 週今日練習題：PostgreSQL 基本 CRUD 操作

## 題目說明

建立一個簡單的「書籍管理 API」，透過 Express.js 搭配 PostgreSQL，實作對 `books` 資料表的完整 CRUD 操作（新增、查詢、更新、刪除）。

---

## 技術要求

- **Express.js**：建立 RESTful 路由，回傳 JSON 格式
- **PostgreSQL**：使用 `pg`（node-postgres）套件連線，並操作 `books` 資料表
- **資料表結構**：
  ```sql
  CREATE TABLE books (
    id     SERIAL PRIMARY KEY,
    title  VARCHAR(255) NOT NULL,
    author VARCHAR(255) NOT NULL,
    year   INTEGER,
    price  NUMERIC(10, 2)
  );
  ```

---

## 具體任務

1. **資料庫連線設定**
   - 安裝 `pg` 套件（`npm install pg`）
   - 建立 `db.js`，使用 `pg.Pool` 建立連線池，並匯出供各路由使用

2. **建立 `books` 資料表**
   - 撰寫 SQL 建立資料表（可放在 `init.sql` 或在伺服器啟動時自動執行）

3. **實作以下四支 API**

   | 方法   | 路徑            | 說明               |
   |--------|----------------|--------------------||
   | POST   | `/books`       | 新增一本書         |
   | GET    | `/books`       | 查詢所有書籍       |
   | PUT    | `/books/:id`   | 更新指定 id 的書   |
   | DELETE | `/books/:id`   | 刪除指定 id 的書   |

4. **POST `/books`**
   - 接收 JSON body：`{ title, author, year, price }`
   - INSERT 進資料庫，回傳新增後的完整資料（含 `id`）與 HTTP 201

5. **GET `/books`**
   - SELECT 全部書籍，依 `id` 升冪排序，回傳陣列

6. **PUT `/books/:id`**
   - 接收 JSON body（可部分欄位）：`{ title?, author?, year?, price? }`
   - UPDATE 對應 id 的資料，回傳更新後的完整資料
   - 若 id 不存在，回傳 404 `{ error: "Book not found" }`

7. **DELETE `/books/:id`**
   - DELETE 對應 id 的資料
   - 若成功，回傳 200 `{ message: "Book deleted" }`
   - 若 id 不存在，回傳 404 `{ error: "Book not found" }`

---

## 提示

- 使用 `pool.query(sql, [params])` 執行 SQL，**務必使用參數化查詢**（`$1`, `$2`...），避免 SQL Injection
- `pg.Pool` 比 `pg.Client` 更適合 Express（自動管理連線）
- `RETURNING *` 是 PostgreSQL 的語法，可讓 INSERT / UPDATE / DELETE 直接回傳受影響的列
  ```sql
  INSERT INTO books (title, author, year, price)
  VALUES ($1, $2, $3, $4)
  RETURNING *;
  ```
- 路由拆分：可將書籍路由放在 `routes/books.js`，再於 `app.js` 用 `app.use('/books', booksRouter)` 掛載

## 進階挑戰（選做）

- **GET `/books/:id`**：查詢單本書籍，若不存在回傳 404
- 使用環境變數（`dotenv`）管理資料庫連線設定（host、port、user、password、database），避免硬編碼
- 新增一支 **GET `/books?author=xxx`** 路由，可依作者名稱過濾書籍（使用 `ILIKE` 做大小寫不敏感搜尋）
- 為所有路由加上統一的錯誤處理 middleware，當 `pool.query` 拋出例外時回傳 500
