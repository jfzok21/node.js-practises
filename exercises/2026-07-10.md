# 第 1 週今日練習題：Express 基礎路由 — 書籍管理 API

## 題目說明

使用 Express.js 建立一個「書籍管理（Books）」REST API，以純記憶體陣列儲存資料（本週不需接資料庫）。實作 GET、POST 兩種 HTTP 方法，並以 JSON 格式回應。透過這道題，你將熟悉 Express 路由的設定方式、JSON 請求體的解析，以及基本的輸入驗證與 HTTP 狀態碼的使用。

---

## 技術要求

- **Express.js**：建立 HTTP 伺服器、定義路由（`app.get`、`app.post`）、解析 JSON 請求體（`express.json()`）
- **PostgreSQL**：本週不使用，資料以 JS 陣列暫存即可

---

## 具體任務

1. **初始化專案**
   - 在 `week1/books/` 資料夾下執行 `npm init -y`，安裝 `express`
   - 建立 `index.js` 作為主程式進入點

2. **啟動 Express 伺服器**
   - 監聽埠號 `3001`
   - 伺服器啟動後印出 `Books API running on http://localhost:3001`

3. **預設資料**
   在記憶體陣列中預先放入以下兩筆書籍：
   ```json
   [
     { "id": 1, "title": "深入淺出 Node.js", "author": "朴靈", "year": 2014, "available": true },
     { "id": 2, "title": "JavaScript 設計模式", "author": "Addy Osmani", "year": 2023, "available": false }
   ]
   ```

4. **GET `/books`**
   - 回傳所有書籍清單：
     ```json
     {
       "count": 2,
       "books": [ ... ]
     }
     ```
   - 支援查詢字串 `?available=true`，只回傳 `available` 為 `true` 的書籍

5. **GET `/books/:id`**
   - 依 `id` 取得單筆書籍
   - 找到時以 HTTP 200 回傳書籍物件
   - 找不到時回傳 **HTTP 404**：
     ```json
     { "error": "找不到 id 為 99 的書籍" }
     ```

6. **POST `/books`**
   - 接收 JSON 請求體：
     ```json
     { "title": "...", "author": "...", "year": 2025 }
     ```
   - `id` 自動遞增，`available` 預設為 `true`
   - 新增成功回傳 **HTTP 201** 及新書籍物件
   - 若 `title` 或 `author` 未提供或為空字串，回傳 **HTTP 400**：
     ```json
     { "error": "title 與 author 為必填欄位" }
     ```

---

## 提示

- 記得在所有路由前加上 `app.use(express.json())`，否則 `req.body` 會是 `undefined`
- 讀取查詢字串使用 `req.query.available`，值為字串 `"true"` 或 `"false"`，需用 `=== 'true'` 比較
- `req.params.id` 是字串，比較 id 時記得轉型：`Number(req.params.id)` 或 `parseInt(req.params.id, 10)`
- 可用 `books.filter(b => b.available === true)` 來篩選可借閱書籍

---

## 進階挑戰（選做）

- **PUT `/books/:id`**：接收完整書籍物件更新指定書籍，回傳更新後的結果（找不到則 404）
- **DELETE `/books/:id`**：刪除指定書籍，成功回傳 **HTTP 204 No Content**
- **GET `/books?author=朴靈`**：支援同時以 `author` 查詢字串過濾書籍，並可與 `available` 一起組合使用
- 使用 `curl` 或 Postman 逐一測試所有路由，並記錄請求與回應範例
