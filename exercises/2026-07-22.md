# 第 3 週今日練習題：書籍搜尋 API — 路由參數與查詢字串

## 題目說明

建立一個書籍管理 REST API，練習使用 Express 的路由參數（`:id`）與查詢字串（`?search=`、`?page=`、`?sort=`）來取得、篩選、搜尋資料庫中的書籍資料。

---

## 技術要求

- **Express.js**：路由參數（`req.params`）、查詢字串（`req.query`）、路由分組（`express.Router`）
- **PostgreSQL**：`WHERE`、`ILIKE`、`ORDER BY`、`LIMIT` / `OFFSET` 條件查詢

---

## 具體任務

1. **建立資料表與測試資料**

   在 PostgreSQL 建立 `books` 資料表：

   ```sql
   CREATE TABLE books (
     id        SERIAL PRIMARY KEY,
     title     VARCHAR(200) NOT NULL,
     author    VARCHAR(100) NOT NULL,
     genre     VARCHAR(50),
     price     NUMERIC(8, 2),
     published_at DATE
   );

   INSERT INTO books (title, author, genre, price, published_at) VALUES
     ('深入淺出 Node.js', '朴靈',       '技術',   480, '2013-12-01'),
     ('JavaScript 大全', 'David Flanagan', '技術', 1050, '2020-08-01'),
     ('Clean Code',      'Robert Martin', '技術',  680, '2008-08-01'),
     ('原子習慣',         '詹姆斯·克利爾', '自我成長', 380, '2019-06-01'),
     ('設計的法則',       '威廉·立德威',   '設計',   520, '2011-01-01');
   ```

2. **實作以下 API 路由**

   | 方法 | 路徑                  | 說明                         |
   |------|-----------------------|------------------------------|
   | GET  | `/api/books`          | 取得所有書籍（支援查詢字串）  |
   | GET  | `/api/books/:id`      | 依 ID 取得單一書籍            |
   | GET  | `/api/books/genre/:genre` | 依分類取得書籍列表        |

3. **`GET /api/books` 支援以下查詢字串**

   - `?search=關鍵字`：對 `title` 與 `author` 做不分大小寫的模糊搜尋（`ILIKE`）
   - `?genre=分類`：篩選特定分類
   - `?sort=price_asc` 或 `?sort=price_desc`：依價格排序（預設不排序）
   - `?min_price=數字&max_price=數字`：依價格範圍篩選

   範例請求：
   ```
   GET /api/books?search=node&sort=price_desc
   GET /api/books?genre=技術&min_price=500
   ```

4. **`GET /api/books/:id` 處理無效 ID**

   - 若 `:id` 為非數字，回傳 `400 Bad Request`
   - 若書籍不存在，回傳 `404 Not Found`

5. **統一回應格式**

   成功時：
   ```json
   {
     "success": true,
     "count": 3,
     "data": [ { "id": 1, "title": "...", "author": "...", "genre": "...", "price": "480.00" } ]
   }
   ```

   失敗時：
   ```json
   {
     "success": false,
     "message": "找不到 ID 為 99 的書籍"
   }
   ```

---

## 提示

- 使用 `req.params.id` 取得路由參數，並用 `parseInt()` 轉換，再用 `isNaN()` 驗證
- 使用 `req.query.search` 取得查詢字串，記得處理 `undefined`（可用 `?? ''`）
- PostgreSQL 的 `ILIKE` 可做不區分大小寫的模糊比對，例如 `title ILIKE $1`，傳入 `%keyword%`
- 動態組裝 SQL 時，建議用陣列收集 `WHERE` 條件字串，再用 `AND` 串接，避免 SQL 注入仍需用參數化查詢
- 可以在 `pg` 查詢時使用 `$1, $2, ...` 佔位符搭配 `values` 陣列

## 進階挑戰（選做）
