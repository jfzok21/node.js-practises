# 第 3 週今日練習題：書籍查詢 API（路由參數與查詢字串）

## 題目說明

建立一個書籍管理 REST API，練習在 Express 中使用動態路由參數（`:id`）與 URL 查詢字串（`?search=&sort=&limit=`）來查詢 PostgreSQL 資料庫中的書籍資料。

## 技術要求

- **Express.js**：使用路由參數 `:id` 取得單筆資料；使用 `req.query` 解析查詢字串做篩選與排序
- **PostgreSQL**：使用 `pg` 套件執行參數化查詢（`$1`, `$2`），防止 SQL Injection；用 `ILIKE` 做模糊搜尋

## 具體任務

1. **建立 `books` 資料表**
   ```sql
   CREATE TABLE IF NOT EXISTS books (
     id SERIAL PRIMARY KEY,
     title VARCHAR(200) NOT NULL,
     author VARCHAR(100) NOT NULL,
     genre VARCHAR(50),
     published_year INTEGER,
     created_at TIMESTAMP DEFAULT NOW()
   );
   ```
   並插入至少 5 筆測試資料。

2. **實作 `GET /books`**
   支援以下查詢字串：
   - `?search=關鍵字` → 用 `ILIKE` 模糊搜尋 `title` 或 `author`
   - `?genre=小說` → 篩選特定分類
   - `?sort=published_year&order=desc` → 排序（欄位與方向由查詢字串決定）
   - `?limit=5&offset=0` → 限制筆數（為下週分頁做準備）
   - 回傳格式：`{ "data": [...], "count": N }`

3. **實作 `GET /books/:id`**
   - 若找到書籍，回傳 `{ "data": { ...book } }`
   - 若 `id` 不是數字，回傳 `400 Bad Request`
   - 若查無資料，回傳 `404 Not Found`

4. **實作 `GET /books/:id/related`**
   根據該書的 `genre`，回傳同分類的其他書籍（最多 3 本，排除自身）。

5. **測試所有端點**
   使用 `curl` 或 Postman 驗證以下情境：
   - `GET /books?search=harry`
   - `GET /books?genre=奇幻&sort=published_year&order=asc`
   - `GET /books/1`
   - `GET /books/999`（預期 404）
   - `GET /books/abc`（預期 400）
   - `GET /books/1/related`

## 提示

- 排序欄位（`sort`）應使用白名單驗證，避免使用者傳入任意 SQL，例如：
  ```js
  const ALLOWED_SORT_COLUMNS = ['title', 'author', 'published_year', 'created_at'];
  const sortColumn = ALLOWED_SORT_COLUMNS.includes(req.query.sort) ? req.query.sort : 'created_at';
  ```
- 查詢字串的值預設都是字串，記得用 `parseInt()` 轉換 `limit`、`offset`、`id`。
- 用 `isNaN()` 或正則表達式檢查 `:id` 是否為有效數字。
- 善用 `async/await` 和 `try/catch` 包住資料庫呼叫。

## 進階挑戰（選做）

- **多欄位排序**：支援 `?sort=published_year,title`（逗號分隔多欄位）。
- **欄位篩選**：支援 `?fields=id,title,author` 只回傳指定欄位（需動態組 SQL SELECT 欄位）。
- **全文搜尋**：改用 PostgreSQL 的 `to_tsvector` / `to_tsquery` 實作更強大的全文搜尋，取代 `ILIKE`。
- **ETag 快取**：為 `GET /books/:id` 加上 `ETag` 與 `If-None-Match` 支援，當資料未改變時回傳 `304 Not Modified`。
