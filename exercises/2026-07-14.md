# 第 1 週今日練習題：打造你的第一個 Express JSON API

## 題目說明

建立一個簡單的 Express 伺服器，提供「書單管理」功能。使用者可以透過 HTTP 請求取得書單列表，也可以新增書籍。所有資料暫時存放在記憶體中的陣列（不需要資料庫），重點在熟悉 Express 路由設置與 JSON 回應格式。

---

## 技術要求

- **Express.js**：建立伺服器、定義 GET 與 POST 路由、回傳 JSON 格式回應
- **Node.js 內建模組**：不需要額外資料庫，使用 JavaScript 陣列模擬資料儲存

---

## 具體任務

1. **初始化專案**
   - 建立新資料夾 `book-api/`，執行 `npm init -y`
   - 安裝 `express`（`npm install express`）
   - 建立 `index.js` 作為入口點，伺服器監聽 port `3000`

2. **建立資料**
   - 在 `index.js` 頂部定義一個陣列 `books`，預先放入 3 筆書籍資料，每筆包含：
     ```js
     { id: 1, title: '書名', author: '作者', year: 2024 }
     ```

3. **實作 GET `/api/books`**
   - 回傳全部書籍的 JSON 陣列
   - 回應格式範例：
     ```json
     { "success": true, "data": [ ... ] }
     ```

4. **實作 GET `/api/books/:id`**
   - 依 `id` 找到對應書籍並回傳
   - 若找不到，回傳 `404` 狀態碼與錯誤訊息：
     ```json
     { "success": false, "message": "找不到此書籍" }
     ```

5. **實作 POST `/api/books`**
   - 從 request body 接收 `title`、`author`、`year`
   - 自動生成新的 `id`（可用 `books.length + 1` 或目前最大 id + 1）
   - 將新書籍推入 `books` 陣列
   - 回傳 `201` 狀態碼與新建立的書籍資料：
     ```json
     { "success": true, "data": { "id": 4, "title": "...", "author": "...", "year": 2025 } }
     ```

6. **測試你的 API**
   - 使用 `curl` 或 Postman / Thunder Client 測試以上三個端點
   - 確認 GET 列表、GET 單本、POST 新增均正常運作

---

## 提示

- Express 要解析 JSON request body，記得加上：
  ```js
  app.use(express.json());
  ```
- 路由參數 `req.params.id` 取到的是**字串**，比對前記得用 `Number(req.params.id)` 或 `parseInt` 轉型
- POST 的 body 欄位從 `req.body.title`、`req.body.author`、`req.body.year` 取得
- 測試 POST 的 curl 指令範例：
  ```bash
  curl -X POST http://localhost:3000/api/books \
    -H "Content-Type: application/json" \
    -d '{"title":"Node.js 入門","author":"小明","year":2025}'
  ```
