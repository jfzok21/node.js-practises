# 第 1 週今日練習題：Express 基礎路由與 JSON 回應

## 題目說明

建立一個簡單的 Express 伺服器，實作多個基礎路由，練習處理 GET 與 POST 請求，並以標準化的 JSON 格式回應客戶端。這是後端 API 開發的基礎，請確保你理解每個路由的職責與回應結構。

---

## 技術要求

- **Express.js**：建立 HTTP 伺服器、定義 GET / POST 路由、回傳 JSON 回應
- **PostgreSQL**：（本週不需要，下週開始引入）

---

## 具體任務

1. **初始化專案**
   - 建立新資料夾 `week1-express-basic`
   - 執行 `npm init -y`，安裝 `express`
   - 建立 `index.js` 作為入口點

2. **建立 Express 伺服器**
   - 監聽 Port `3000`
   - 啟動後在 console 輸出 `Server is running on http://localhost:3000`

3. **實作以下路由**

   | Method | Path         | 功能說明                                      |
   |--------|-------------|---------------------------------------------|
   | GET    | `/`          | 回傳歡迎訊息 JSON                              |
   | GET    | `/ping`      | 回傳 `{ status: "ok", message: "pong" }`     |
   | GET    | `/users`     | 回傳一組假的使用者陣列（至少 3 筆資料）         |
   | POST   | `/users`     | 接收 JSON body（name、email），回傳新建立的使用者資料 |
   | GET    | `/status`    | 回傳伺服器狀態，包含當前時間（`new Date()`）    |

4. **統一 JSON 回應格式**
   所有成功回應都使用以下結構：
   ```json
   {
     "success": true,
     "data": { ... }
   }
   ```
   POST 路由失敗（缺少欄位）時回傳：
   ```json
   {
     "success": false,
     "message": "name 和 email 為必填欄位"
   }
   ```

5. **啟用 JSON 解析**
   使用 `express.json()` middleware，讓 POST 請求可以讀取 `req.body`

---

## 提示

- `app.use(express.json())` 必須放在所有路由定義之前
- 使用 `res.status(201).json(...)` 回傳 POST 新建資源的成功回應
- 使用 `res.status(400).json(...)` 回傳驗證失敗的錯誤回應
- 可用 [Postman](https://www.postman.com/) 或 `curl` 測試 POST 路由：
  ```bash
  curl -X POST http://localhost:3000/users \
    -H "Content-Type: application/json" \
    -d '{"name": "Alice", "email": "alice@example.com"}'
  ```
- GET 路由可直接在瀏覽器打開測試

---

## 進階挑戰（選做）

- 為 `/users` GET 路由加入簡單的 query string 篩選：`?name=Alice`，只回傳 name 匹配的使用者
- 新增 `GET /users/:id` 路由，根據假資料的 id 回傳單一使用者，找不到時回傳 `404`
- 將假資料抽成獨立的 `data/users.js` 模組，讓 `index.js` 引入使用（練習模組化）
