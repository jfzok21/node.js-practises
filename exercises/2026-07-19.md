# 第 2 週今日練習題：Express + PostgreSQL 基本 CRUD 操作

## 題目說明

建立一個簡單的「書籍管理 API」，透過 Express.js 串接 PostgreSQL 資料庫，實作完整的 CRUD（新增、查詢、更新、刪除）功能。本練習的重點在於學習如何建立資料庫連線、設計資料表，以及透過 SQL 語句操作資料。

---

## 技術要求

- **Express.js**：建立 RESTful API 路由，處理 HTTP 請求與回應
- **PostgreSQL**：使用 `pg`（node-postgres）套件連線資料庫，執行 SQL 語句

---

## 具體任務

### 1. 初始化專案與安裝依賴

```bash
npm init -y
npm install express pg dotenv
```

建立 `.env` 檔案（不要 commit 此檔案）：

```
DB_HOST=localhost
DB_PORT=5432
DB_NAME=bookstore
DB_USER=postgres
DB_PASSWORD=yourpassword
PORT=3000
```

### 2. 建立 PostgreSQL 資料表

連線到 PostgreSQL，建立資料庫與資料表：

```sql
CREATE DATABASE bookstore;

\c bookstore

CREATE TABLE books (
  id SERIAL PRIMARY KEY,
  title VARCHAR(255) NOT NULL,
  author VARCHAR(255) NOT NULL,
  price NUMERIC(10, 2) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 3. 建立資料庫連線模組

建立 `db.js`，使用 `pg.Pool` 管理連線池：

```js
// db.js
const { Pool } = require('pg');
require('dotenv').config();

const pool = new Pool({
  host: process.env.DB_HOST,
  port: process.env.DB_PORT,
  database: process.env.DB_NAME,
  user: process.env.DB_USER,
  password: process.env.DB_PASSWORD,
});

module.exports = pool;
```

### 4. 實作以下 API 端點

| 方法   | 路徑            | 功能             |
|--------|-----------------|------------------|
| GET    | `/books`        | 取得所有書籍     |
| GET    | `/books/:id`    | 取得單本書籍     |
| POST   | `/books`        | 新增一本書籍     |
| PUT    | `/books/:id`    | 更新書籍資料     |
| DELETE | `/books/:id`    | 刪除一本書籍     |

**POST /books 的 request body 格式：**

```json
{
  "title": "Learning Node.js",
  "author": "Marc Wandschneider",
  "price": 450.00
}
```

**所有回應應使用統一格式：**

```json
{
  "success": true,
  "data": { ... }
}
```

錯誤時：

```json
{
  "success": false,
  "message": "Book not found"
}
```

### 5. 測試所有端點

使用 `curl` 或 Postman 依序測試：

1. 新增 3 本書（POST）
2. 查詢所有書籍（GET /books）
3. 查詢單本書籍（GET /books/1）
4. 更新第一本書的價格（PUT /books/1）
5. 刪除第三本書（DELETE /books/3）
6. 確認刪除後查詢結果（GET /books）

---

## 提示

- 使用 `pool.query(sql, [params])` 的參數化查詢（`$1`, `$2`...）防止 SQL Injection
- `INSERT ... RETURNING *` 可讓 INSERT 語句同時回傳剛新增的資料
- 當 SELECT 結果為空（查無資料）時，`rows` 陣列長度為 0，記得回傳 404
- 在 `app.use(express.json())` 之後才能讀取 `req.body`
- 可以先用 `pool.query('SELECT NOW()')` 確認資料庫連線是否成功

---

## 進階挑戰（選做）

1. **連線錯誤處理**：若資料庫連線失敗，API 應回傳有意義的錯誤訊息，而非讓程式崩潰
2. **輸入驗證**：POST / PUT 時，若 `title`、`author`、`price` 任一欄位缺失，回傳 400 錯誤
3. **價格範圍篩選**：新增 `GET /books?minPrice=100&maxPrice=500` 功能，透過 WHERE 子句篩選書籍
4. **軟刪除（Soft Delete）**：在資料表新增 `deleted_at` 欄位，DELETE 時僅更新此欄位而非真正刪除資料，查詢時過濾掉已刪除的資料
