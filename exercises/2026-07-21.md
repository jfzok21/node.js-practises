# 第 2 週今日練習題：PostgreSQL 基本 CRUD 操作

## 題目說明

建立一個「書籍管理 API」，使用 Express.js 搭配 PostgreSQL，實作完整的新增、讀取、更新、刪除（CRUD）功能。透過這個練習，你將學會如何讓 Express 應用程式連線到 PostgreSQL 資料庫，並以 SQL 指令操作資料。

---

## 技術要求

- **Express.js**：建立 RESTful API 路由，處理 HTTP 請求與回應
- **PostgreSQL**：使用 `pg`（node-postgres）套件建立連線池，執行 INSERT / SELECT / UPDATE / DELETE 查詢
- **資料表結構**：
  ```sql
  CREATE TABLE books (
    id        SERIAL PRIMARY KEY,
    title     VARCHAR(255) NOT NULL,
    author    VARCHAR(255) NOT NULL,
    price     NUMERIC(10, 2) NOT NULL,
    created_at TIMESTAMPTZ DEFAULT NOW()
  );
  ```

---

## 具體任務

1. **環境設定**
   - 安裝必要套件：`npm install express pg dotenv`
   - 建立 `.env` 檔案，設定 `DATABASE_URL`（或 `DB_HOST`、`DB_USER`、`DB_PASSWORD`、`DB_NAME`、`DB_PORT`）
   - 建立 `db.js`，使用 `pg.Pool` 建立連線池並匯出

2. **建立資料表**
   - 撰寫 `init.sql`，包含 `CREATE TABLE IF NOT EXISTS books (...)` 語句
   - 在應用程式啟動時或手動執行此 SQL 初始化資料表

3. **實作 API 端點**
   - `POST   /books`       — 新增一本書（INSERT）
   - `GET    /books`       — 取得所有書籍（SELECT）
   - `GET    /books/:id`   — 依 ID 取得單本書（SELECT WHERE id = $1）
   - `PUT    /books/:id`   — 更新書籍資料（UPDATE）
   - `DELETE /books/:id`   — 刪除書籍（DELETE）

4. **Request / Response 格式**
   - 請求 Body（POST / PUT）：
     ```json
     { "title": "Node.js 實戰", "author": "王小明", "price": 450 }
     ```
   - 成功回應：HTTP 200 / 201，回傳 JSON 物件或陣列
   - 找不到資源：HTTP 404，回傳 `{ "error": "Book not found" }`

5. **測試 API**
   - 使用 `curl` 或 Postman 依序測試所有端點
   - 確認 INSERT 後可透過 GET 查詢到新資料
   - 確認 UPDATE 後資料確實變更
   - 確認 DELETE 後再 GET 回傳 404

---

## 提示

- 使用 **參數化查詢**（`$1`, `$2`...）避免 SQL Injection，例如：
  ```js
  const result = await pool.query('SELECT * FROM books WHERE id = $1', [id]);
  ```
- `pool.query()` 回傳物件的 `rows` 屬性為結果陣列，`rowCount` 為影響行數
- 判斷 UPDATE / DELETE 是否找到目標：`result.rowCount === 0` 時回傳 404
- 把 `pool` 放在獨立的 `db.js` 模組，讓各路由都可以 `require('./db')` 引用
- `.env` 檔不要 commit 進 git，記得加入 `.gitignore`

## 進階挑戰（選做）

1. **輸入驗證**：在 POST / PUT 時檢查 `title`、`author`、`price` 是否存在且型別正確，若不合法回傳 HTTP 400 與錯誤訊息
2. **搜尋功能**：新增 `GET /books?author=王小明` 端點，使用 `ILIKE` 模糊比對作者姓名
3. **排序**：`GET /books?sort=price&order=asc` 讓呼叫者指定排序欄位與方向（注意：欄位名稱需白名單過濾，不可直接拼入 SQL）
4. **連線錯誤處理**：在 `db.js` 加入 `pool.on('error', ...)` 監聽非預期的連線中斷事件
