# 今日練習題：PostgreSQL 書籍資料庫 CRUD

## 題目說明
建立一個簡單的書籍管理系統，使用 Node.js 搭配 `pg` 套件連接 PostgreSQL 資料庫。你需要建立一張 `books` 資料表，並實作完整的 CRUD 操作（新增、查詢、修改、刪除）。本練習的重點在於熟悉 PostgreSQL 的基本語法、參數化查詢的寫法，以及如何透過 Node.js 程式操作資料庫。完成後，你將能夠透過程式碼管理資料庫中的書籍資料，包含書名、作者、價格與庫存數量。

## 技術要求
- **Express.js**：建立 RESTful 路由，提供 HTTP 介面操作資料庫
- **PostgreSQL**：`CREATE TABLE`、`INSERT`、`SELECT`、`UPDATE`、`DELETE` 語法；參數化查詢（`$1`, `$2`）防止 SQL Injection

## 具體任務
1. 建立 `books` 資料表，包含欄位：`id`（SERIAL PRIMARY KEY）、`title`（VARCHAR）、`author`（VARCHAR）、`price`（NUMERIC）、`stock`（INTEGER），並在資料庫中執行建立語法
2. 撰寫 `POST /books` 路由，接收 JSON 格式的書籍資料（title、author、price、stock），使用 `INSERT INTO` 將資料存入資料庫，並回傳新增的完整記錄
3. 撰寫 `GET /books` 路由，使用 `SELECT * FROM books` 查詢所有書籍並以 JSON 陣列回傳；另外撰寫 `GET /books/:id` 路由，依 id 查詢單筆資料
4. 撰寫 `PUT /books/:id` 路由，接收要更新的欄位（可部分更新），使用 `UPDATE ... WHERE id = $1` 修改指定書籍資料並回傳更新後的記錄
5. 撰寫 `DELETE /books/:id` 路由，使用 `DELETE FROM books WHERE id = $1` 刪除指定書籍，並回傳 `{ message: "已刪除" }` 確認訊息

## 提示
- 使用 `pg` 套件的 `Pool` 建立資料庫連線池，連線設定放在獨立的 `db.js` 檔案中方便管理
- 所有 SQL 查詢務必使用參數化查詢（`$1`, `$2` 佔位符），避免直接拼接字串造成 SQL Injection 漏洞
- 善用 `async/await` 搭配 `try/catch` 處理資料庫操作的非同步錯誤，當查詢失敗時回傳 HTTP 500 與錯誤訊息

## 進階挑戰（選做）
在 `GET /books` 路由加入查詢字串支援，例如 `GET /books?author=村上春樹` 可依作者篩選，`GET /books?sort=price&order=asc` 可依價格排序，讓查詢更具彈性。
