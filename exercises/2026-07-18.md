# 第 2 週今日練習題：使用 Express 操作 PostgreSQL 基本 CRUD

## 題目說明

建立一個簡單的「書籍管理 API」，透過 Express.js 連接 PostgreSQL 資料庫，實作完整的 CRUD 功能（新增、查詢、更新、刪除）。

## 技術要求

- **Express.js**：建立 RESTful API 路由，回傳 JSON 格式資料
- **PostgreSQL**：使用 `pg`（node-postgres）套件進行資料庫連線與操作

## 具體任務

1. **環境設定**
   - 安裝所需套件：`express`、`pg`、`dotenv`
   - 建立 `.env` 檔案，設定資料庫連線資訊（`DB_HOST`、`DB_PORT`、`DB_USER`、`DB_PASSWORD`、`DB_NAME`）
   - 建立 `db.js` 模組，匯出 `pg.Pool` 供各路由使用

2. **建立資料表**
   在 PostgreSQL 中執行以下 SQL，建立 `books` 資料表：
   ```sql
   CREATE TABLE books (
     id SERIAL PRIMARY KEY,
     title VARCHAR(255) NOT NULL,
     author VARCHAR(255) NOT NULL,
     published_year INTEGER,
     created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
   );
   ```

3. **實作以下 API 端點**

   | 方法   | 路徑          | 功能               |
   |--------|---------------|-----------------|
   | GET    | /books        | 取得所有書籍列表   |
   | GET    | /books/:id    | 取得單筆書籍資料   |
   | POST   | /books        | 新增一本書         |
   | PUT    | /books/:id    | 更新書籍資料       |
   | DELETE | /books/:id    | 刪除一本書         |

4. **回傳格式規範**
   - 成功時回傳適當的 HTTP 狀態碼（200、201、204）
   - 找不到資料時回傳 `404` 與 `{ "error": "Book not found" }`
   - 新增成功時回傳 `201` 與新建立的書籍物件

## 提示

- 使用 `pool.query(sql, [params])` 執行參數化查詢，避免 SQL Injection
- `INSERT ... RETURNING *` 可讓 PostgreSQL 直接回傳剛新增的資料列
- 用 `rows[0]` 取得單筆結果；若 `rows.length === 0`，代表查無資料，應回傳 404
- 記得在 Express 中加上 `express.json()` middleware，才能解析 POST/PUT 的 request body
- 可用 [Postman](https://www.postman.com/) 或 `curl` 測試各端點

## 進階挑戰（選做）

- 為 `POST /books` 加入基本驗證：`title` 與 `author` 為必填，缺少時回傳 `400 Bad Request`
- 新增 `GET /books?author=村上春樹` 查詢功能，依作者名稱篩選書籍（使用 `ILIKE` 做不區分大小寫搜尋）
- 將資料庫操作抽離至獨立的 `books.repository.js` 模組，讓路由層保持簡潔
