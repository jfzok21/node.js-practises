# 第 1 週今日練習題：打造你的第一個 Express JSON API

## 題目說明

建立一個簡單的 Express 應用程式，提供 RESTful 風格的 JSON 回應端點。練習的重點在於理解 Express 路由的基本結構，以及如何正確回傳 JSON 格式的資料給客戶端。

## 技術要求
- **Express.js**：使用 `express()` 建立應用程式；設定路由處理 `GET` 與 `POST` 請求；使用 `res.json()` 回傳 JSON 資料；監聽在 `PORT=3000`
- **PostgreSQL**：本週暫不連接資料庫，資料可暫時存放在記憶體（陣列）中

## 具體任務

1. **初始化專案**
   - 在 `exercises/week1/` 目錄下建立 `app.js`
   - 執行 `npm init -y` 並安裝 `express`

2. **建立以下 GET 路由**
   - `GET /` → 回傳 `{ "message": "Hello, Express!" }`
   - `GET /health` → 回傳 `{ "status": "ok", "timestamp": "<現在的 ISO 時間>" }`
   - `GET /users` → 回傳一個包含至少 3 筆假資料的使用者陣列，每筆格式為 `{ "id": 1, "name": "...", "email": "..." }`

3. **建立以下 POST 路由**
   - `POST /users` → 從 `req.body` 讀取 `name` 與 `email`，將新使用者加入記憶體陣列，並回傳 `{ "message": "使用者已建立", "user": { ... } }`（HTTP 狀態碼 201）

4. **測試所有端點**
   - 使用 `curl` 或 Postman 測試所有路由
   - 確認 `POST /users` 之後再呼叫 `GET /users` 可看到新增的資料

## 提示

- 記得加上 `express.json()` middleware，否則 `req.body` 會是 `undefined`：
  ```js
  app.use(express.json());
  ```
- 回傳 201 狀態碼的寫法：`res.status(201).json({ ... })`
- 記憶體陣列範例：
  ```js
  let users = [
    { id: 1, name: 'Alice', email: 'alice@example.com' },
    { id: 2, name: 'Bob',   email: 'bob@example.com' },
  ];
  ```
- 新使用者的 `id` 可以用 `users.length + 1` 或 `Date.now()` 產生

## 進階挑戰（選做）

- 加入 `GET /users/:id` 路由，依 id 回傳單一使用者；若不存在則回傳 `{ "error": "找不到使用者" }`（HTTP 404）
- 加入 `DELETE /users/:id` 路由，從記憶體陣列中移除指定使用者，回傳 `{ "message": "使用者已刪除" }`
- 嘗試將假資料改為從外部 JSON 檔案（`data/users.json`）讀取，體驗資料與邏輯分離的感覺
