# 第 2 週今日 Node.js 練習題：非同步檔案管理系統

## 題目說明

建立一個簡單的「檔案管理工具」，能夠對本地檔案進行讀取、寫入、附加內容、刪除，並列出指定目錄下的所有檔案，使用 Node.js 內建的 `fs` 模組完成所有操作。

---

## 技術要求

- **概念**：fs.readFile、fs.writeFile、fs.appendFile、fs.unlink、fs.readdir、fs.promises（Promise API）
- **需要的套件**：無（使用 Node.js 內建 `fs` 模組）

---

## 具體任務

1. **讀取檔案**：使用 `fs.promises.readFile` 讀取一個文字檔，並將內容印出至 console。
2. **寫入檔案**：使用 `fs.promises.writeFile` 將一段文字寫入新檔案（若檔案已存在則覆蓋）。
3. **附加內容**：使用 `fs.promises.appendFile` 將新的一行文字附加到現有檔案末尾。
4. **列出目錄**：使用 `fs.promises.readdir` 列出目前目錄下所有檔案名稱。
5. **刪除檔案**：使用 `fs.promises.unlink` 刪除指定檔案，並在刪除前確認檔案存在（使用 `fs.promises.access`）。

---

## 範例

```js
const fs = require('fs').promises;
const path = require('path');

async function main() {
  const filePath = path.join(__dirname, 'demo.txt');

  // 1. 寫入檔案
  await fs.writeFile(filePath, '第一行：Hello, Node.js!\n', 'utf8');
  console.log('✅ 寫入成功');

  // 2. 附加內容
  await fs.appendFile(filePath, '第二行：fs 模組很好用！\n', 'utf8');
  console.log('✅ 附加成功');

  // 3. 讀取檔案
  const content = await fs.readFile(filePath, 'utf8');
  console.log('📄 檔案內容：');
  console.log(content);

  // 4. 列出目錄
  const files = await fs.readdir(__dirname);
  console.log('📁 目前目錄檔案：', files);

  // 5. 刪除前確認存在
  try {
    await fs.access(filePath);
    await fs.unlink(filePath);
    console.log('🗑️ 刪除成功');
  } catch {
    console.log('⚠️ 檔案不存在，無法刪除');
  }
}

main().catch(console.error);
```

**預期輸出：**
```
✅ 寫入成功
✅ 附加成功
📄 檔案內容：
第一行：Hello, Node.js!
第二行：fs 模組很好用！

📁 目前目錄檔案：[ 'demo.txt', 'index.js', ... ]
🗑️ 刪除成功
```

---

## 提示

- `require('fs').promises` 與 `require('fs/promises')` 效果相同（Node.js 14+）。
- `fs.access` 只是確認「可否存取」，若無法存取會丟出錯誤，善用 try/catch 處理。
- `path.join(__dirname, 'filename')` 可避免跨平台路徑問題（Windows 用 `\`，Unix 用 `/`）。
- 所有 `fs.promises` 的函式都回傳 Promise，記得搭配 `await` 使用。

## 學習重點（跟後端 API 的關聯）

後端 API 中，`fs` 模組的應用無所不在：

| 應用場景 | 說明 |
|---|---|
| 讀取設定檔 | 啟動時讀取 `config.json` 或 `.env` 載入環境設定 |
| 寫入 Log | 將 API 請求紀錄寫入日誌檔，便於除錯 |
| 檔案上傳 | 接收用戶上傳後，用 `fs` 儲存至伺服器目錄 |
| 靜態資源服務 | 讀取 HTML / CSS / JS 檔案回傳給用戶端 |

掌握 `fs` 模組後，第 11 週學習 `multer` 檔案上傳時會更加順手。
