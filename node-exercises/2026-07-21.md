# 第 3 週今日 Node.js 練習題：非同步任務排程器

## 題目說明

實作一個簡單的「非同步任務排程器」，分別用 **callback**、**Promise**、**async/await** 三種寫法，模擬從資料庫讀取使用者資料、驗證權限、再回傳結果的流程。透過這個練習，你將深刻理解 Node.js 非同步程式設計的演進歷程，以及各寫法的優缺點。

## 技術要求

- **概念**：callback hell、Promise chaining、async/await、錯誤處理（try/catch vs .catch()）
- **需要的套件**：無（純 Node.js 內建）

## 具體任務

1. **Callback 版本**
   - 建立 `getUser(id, callback)` 函式，使用 `setTimeout` 模擬非同步延遲（300ms）後回傳假使用者物件 `{ id, name, role }`
   - 建立 `checkPermission(user, callback)` 函式，延遲 200ms 後判斷 `user.role === 'admin'`，若否則以 `callback(new Error('權限不足'))` 回傳錯誤
   - 建立 `fetchData(userId, callback)` 函式，依序呼叫上兩個函式（巢狀 callback），最後印出 `資料已取得：[name]`

2. **Promise 版本**
   - 將 `getUser` 與 `checkPermission` 改寫為回傳 Promise 的函式
   - 用 `.then().catch()` 串接完成相同流程

3. **Async/Await 版本**
   - 將步驟 2 的 Promise 函式包在 `async function run()` 中
   - 使用 `try/catch` 處理錯誤
   - 測試兩種情境：userId = `1`（admin，應成功）、userId = `2`（一般用戶，應拋錯）

4. **執行並觀察**
   - 在同一支 `index.js` 依序執行三個版本，觀察輸出順序與錯誤訊息格式的差異

## 範例

```js
// 模擬假資料庫
const users = {
  1: { id: 1, name: 'Alice', role: 'admin' },
  2: { id: 2, name: 'Bob',   role: 'user'  },
};

// --- Callback 版本 ---
function getUser(id, callback) {
  setTimeout(() => {
    const user = users[id];
    if (!user) return callback(new Error('使用者不存在'));
    callback(null, user);
  }, 300);
}

function checkPermission(user, callback) {
  setTimeout(() => {
    if (user.role !== 'admin') return callback(new Error('權限不足'));
    callback(null, true);
  }, 200);
}

function fetchData(userId, callback) {
  getUser(userId, (err, user) => {
    if (err) return callback(err);
    checkPermission(user, (err) => {
      if (err) return callback(err);
      callback(null, `資料已取得：${user.name}`);
    });
  });
}

fetchData(1, (err, result) => {
  if (err) console.error('[Callback] 錯誤：', err.message);
  else console.log('[Callback]', result);
});

// --- Promise 版本 ---
function getUserP(id) {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      const user = users[id];
      user ? resolve(user) : reject(new Error('使用者不存在'));
    }, 300);
  });
}

function checkPermissionP(user) {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      user.role === 'admin' ? resolve(user) : reject(new Error('權限不足'));
    }, 200);
  });
}

getUserP(2)
  .then(checkPermissionP)
  .then(user => console.log('[Promise] 資料已取得：', user.name))
  .catch(err => console.error('[Promise] 錯誤：', err.message));

// --- Async/Await 版本 ---
async function run(userId) {
  try {
    const user = await getUserP(userId);
    await checkPermissionP(user);
    console.log('[Async/Await] 資料已取得：', user.name);
  } catch (err) {
    console.error('[Async/Await] 錯誤：', err.message);
  }
}

run(1); // 成功
run(2); // 失敗
```

**預期輸出（順序可能因非同步而略有差異）：**
```
[Callback] 資料已取得：Alice
[Promise] 錯誤：權限不足
[Async/Await] 資料已取得：Alice
[Async/Await] 錯誤：權限不足
```

## 提示

- `callback` 慣例：第一個參數為 `err`，第二個為結果（Node.js error-first convention）
- `new Promise((resolve, reject) => ...)` 中只有第一次呼叫 `resolve` 或 `reject` 會生效
- `async function` 永遠回傳 Promise，可以在外層繼續 `.then()` 或 `await` 它
- 若想看到清楚的執行順序，可在每個步驟加上 `console.log` 時間戳記：`new Date().toISOString()`

## 學習重點

**與後端 API 的關聯：**

在真實的 Express API 中，幾乎所有操作都是非同步的：讀取資料庫、呼叫第三方 API、讀寫檔案。現代 Node.js 後端幾乎全面採用 `async/await`，因為它讓非同步程式碼的閱讀方式接近同步，大幅降低維護成本。這個練習中的「取使用者 → 驗證權限 → 回傳資料」流程，正是後端 middleware 與 route handler 的典型樣式。

## 進階挑戰（選做）

1. **並行執行**：使用 `Promise.all()` 同時發出兩個 `getUserP()` 請求，計算總耗時是否接近單一請求時間（而非兩倍）
2. **超時機制**：實作 `withTimeout(promise, ms)` 包裝函式，若 Promise 在 `ms` 毫秒內未解決則自動 reject 並附上 `'請求逾時'` 錯誤
3. **重試邏輯**：實作 `retry(fn, times)` 函式，當 Promise reject 時最多重試 `times` 次，每次間隔 100ms
