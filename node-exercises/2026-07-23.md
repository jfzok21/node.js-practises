# 第 3 週今日 Node.js 練習題：非同步任務排程器

## 題目說明

建立一個「非同步任務排程器」，透過 Callback、Promise 以及 async/await 三種方式，依序模擬從遠端 API 抓取使用者資料、訂單清單，最後彙整成報表。

## 技術要求
- 概念：callback、Promise chain、async/await、錯誤處理（try/catch）
- 需要的套件：無（使用 Node.js 內建 `setTimeout` 模擬非同步 I/O）

## 具體任務

1. **Callback 版本**：
   - 實作 `fetchUser(id, callback)` — 用 `setTimeout` 模擬 300ms 延遲後，回傳 `{ id, name: 'Alice' }` 或錯誤
   - 實作 `fetchOrders(userId, callback)` — 模擬 200ms 延遲後，回傳訂單陣列
   - 串接兩個函式：先取得使用者，再取得該使用者的訂單，最後印出結果（體驗 callback hell）

2. **Promise 版本**：
   - 將上述兩個函式改寫為回傳 `Promise`
   - 用 `.then().catch()` 串接，取得相同結果

3. **async/await 版本**：
   - 用 `async function` 搭配 `await` 改寫相同流程
   - 加上 `try/catch` 統一處理錯誤
   - 利用 `Promise.all` 同時發送兩個不相依的請求，觀察執行時間差異

## 範例

```js
// --- Callback 版本 ---
function fetchUser(id, callback) {
  setTimeout(() => {
    if (!id) return callback(new Error('缺少 id'));
    callback(null, { id, name: 'Alice' });
  }, 300);
}

function fetchOrders(userId, callback) {
  setTimeout(() => {
    callback(null, [
      { orderId: 1, item: '書本', userId },
      { orderId: 2, item: '鍵盤', userId },
    ]);
  }, 200);
}

// 串接（callback hell 示範）
fetchUser(42, (err, user) => {
  if (err) return console.error('錯誤：', err.message);
  fetchOrders(user.id, (err, orders) => {
    if (err) return console.error('錯誤：', err.message);
    console.log('使用者：', user.name);
    console.log('訂單數：', orders.length);
  });
});

// --- Promise 版本 ---
function fetchUserP(id) {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      if (!id) return reject(new Error('缺少 id'));
      resolve({ id, name: 'Alice' });
    }, 300);
  });
}

function fetchOrdersP(userId) {
  return new Promise((resolve) => {
    setTimeout(() => {
      resolve([
        { orderId: 1, item: '書本', userId },
        { orderId: 2, item: '鍵盤', userId },
      ]);
    }, 200);
  });
}

fetchUserP(42)
  .then((user) => fetchOrdersP(user.id).then((orders) => ({ user, orders })))
  .then(({ user, orders }) => {
    console.log('使用者：', user.name);
    console.log('訂單數：', orders.length);
  })
  .catch((err) => console.error('錯誤：', err.message));

// --- async/await 版本 ---
async function generateReport(userId) {
  try {
    const user = await fetchUserP(userId);
    const orders = await fetchOrdersP(user.id);
    console.log('使用者：', user.name);
    console.log('訂單數：', orders.length);
  } catch (err) {
    console.error('錯誤：', err.message);
  }
}

generateReport(42);

// Promise.all 平行版本
async function generateReportParallel(userId) {
  const [user, allOrders] = await Promise.all([
    fetchUserP(userId),
    fetchOrdersP(userId),   // 假設 userId 已知，兩者可同時發送
  ]);
  console.log('[平行] 使用者：', user.name, '| 訂單數：', allOrders.length);
}

generateReportParallel(42);
```

**預期輸出：**
```
使用者： Alice
訂單數： 2
使用者： Alice
訂單數： 2
使用者： Alice
訂單數： 2
[平行] 使用者： Alice | 訂單數： 2
```

## 提示
- `setTimeout(fn, ms)` 是非同步的，不會阻塞主執行緒
- Promise 的 `reject` 對應 callback 的第一個參數（`err`）
- `async` 函式永遠回傳 Promise，因此可以繼續用 `.then()` 串接
- `Promise.all` 會等待所有 Promise 都 resolve，任一個 reject 就整個失敗
- 計算 `Promise.all` 版本的執行時間：應接近兩者中較長的那個（300ms），而非相加（500ms）

## 學習重點（跟後端 API 的關聯）

在真實的 Express 路由中，你幾乎每個 handler 都會用到 `async/await`：

```js
app.get('/users/:id/report', async (req, res) => {
  try {
    const user = await db.findUser(req.params.id);     // 查 DB（非同步）
    const orders = await db.findOrders(user.id);       // 再查 DB
    res.json({ user, orders });
  } catch (err) {
    res.status(500).json({ error: err.message });
  }
});
```

理解 async/await 讓你能寫出清晰、可維護的路由邏輯，並且正確處理資料庫或外部 API 的錯誤。

## 進階挑戰（選做）

1. **逾時保護**：用 `Promise.race` 讓任一請求超過 250ms 就自動拒絕並回傳 `{ error: 'timeout' }`
2. **重試機制**：實作 `withRetry(fn, maxRetries)` 高階函式，讓失敗的 Promise 自動重試最多 N 次
3. **並行限制**：有 10 個 userId，但一次最多只能同時發送 3 個請求，完成所有請求後彙整結果（提示：可用 semaphore 或遞迴控制）
