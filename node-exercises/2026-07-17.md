# 第 2 週今日 Node.js 練習題：讀寫文字記事本

## 題目說明

請使用 Node.js 內建的 `fs` 模組，打造一個簡易的「文字記事本 CLI 工具」。使用者可以透過命令列參數來新增、列出、刪除記事，所有記事皆以 JSON 格式儲存在本地檔案中。

## 技術要求

- **概念：** `fs.readFileSync`、`fs.writeFileSync`、`fs.existsSync`、JSON 序列化與反序列化、`process.argv`
- **需要的套件：** 無（僅使用 Node.js 內建模組）

## 具體任務

1. 建立 `notes.js`，讀取 `process.argv` 取得使用者的指令（`add`、`list`、`delete`）。
2. 記事資料儲存在 `notes.json`（若不存在則自動建立）；每筆記事結構為 `{ id, text, createdAt }`。
3. 實作 `add <文字>` 指令：新增一筆記事，`id` 以目前記事總數 + 1 計算，`createdAt` 為現在的 ISO 時間字串。
4. 實作 `list` 指令：讀取並印出所有記事，格式為 `[id] text (createdAt)`。
5. 實作 `delete <id>` 指令：從 `notes.json` 中移除指定 `id` 的記事，若找不到則顯示錯誤訊息。
6. 任何寫入操作完成後，輸出操作成功或失敗的提示訊息。

## 範例

```js
// 新增記事
// 執行：node notes.js add "今天學會了 fs 模組"
// 輸出：✅ 已新增記事 [1]：今天學會了 fs 模組

// 再新增一筆
// 執行：node notes.js add "明天要練習 Promise"
// 輸出：✅ 已新增記事 [2]：明天要練習 Promise

// 列出所有記事
// 執行：node notes.js list
// 輸出：
// [1] 今天學會了 fs 模組 (2026-07-17T08:00:00.000Z)
// [2] 明天要練習 Promise (2026-07-17T08:01:00.000Z)

// 刪除記事
// 執行：node notes.js delete 1
// 輸出：🗑️  已刪除記事 [1]

// 刪除不存在的記事
// 執行：node notes.js delete 99
// 輸出：❌ 找不到 id 為 99 的記事
```

## 提示

- 使用 `fs.existsSync(filePath)` 判斷 `notes.json` 是否已存在，不存在時初始化為空陣列 `[]`。
- `JSON.parse(fs.readFileSync(filePath, 'utf8'))` 可一行完成讀取並解析。
- `fs.writeFileSync(filePath, JSON.stringify(data, null, 2))` 寫入時加上 `null, 2` 讓 JSON 排版更易讀。
- `process.argv[2]` 是第一個使用者參數（指令名稱），`process.argv[3]` 是第二個（記事內容或 id）。
- 刪除時用 `Array.prototype.filter` 過濾掉目標 id，再比較長度確認是否真的找到過。

## 學習重點（跟後端 API 的關聯）

在真實的後端 API 開發中，`fs` 的讀寫概念對應的就是資料庫的 **CRUD 操作**：

| 記事本操作 | 後端 API 對應 |
|-----------|--------------|
| `add`     | `POST /notes` |
| `list`    | `GET /notes` |
| `delete`  | `DELETE /notes/:id` |
| `notes.json` | 資料庫（PostgreSQL / MongoDB） |

透過這個練習，你會直覺地理解：資料需要「持久化」（persistence）、操作必須有錯誤處理、回應必須有明確的成功/失敗提示——這些原則在後續學 Express REST API 時完全通用。

## 進階挑戰（選做）

- **挑戰 1：** 新增 `update <id> <新文字>` 指令，允許修改現有記事內容，並加上 `updatedAt` 欄位。
- **挑戰 2：** 新增 `search <關鍵字>` 指令，列出所有包含該關鍵字的記事。
- **挑戰 3：** 使用 `fs.promises`（非同步版本）搭配 `async/await` 重寫所有 `fs` 操作，感受同步與非同步寫法的差異。
- **挑戰 4：** 新增簡單的輸入驗證——`add` 時若文字為空則拒絕新增，`delete` 時若 id 不是數字則顯示提示。
