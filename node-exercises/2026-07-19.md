# 第 2 週今日 Node.js 練習題：讀取與寫入日誌檔案

## 題目說明

建立一個簡單的「日誌管理器」，使用 Node.js 內建的 `fs` 模組來讀取、寫入與附加文字到日誌檔案。這個練習模擬後端 API 伺服器常見的日誌記錄行為。

## 技術要求
- 概念：`fs.readFile`、`fs.writeFile`、`fs.appendFile`、`fs.existsSync`、同步 vs 非同步操作
- 需要的套件：無（使用 Node.js 內建 `fs` 模組）

## 具體任務

1. **初始化日誌**：使用 `fs.writeFile` 建立 `app.log`，若檔案已存在則覆蓋，寫入第一筆記錄（包含當前時間戳記）。
2. **附加日誌**：使用 `fs.appendFile` 在 `app.log` 追加三筆不同的日誌訊息（例如：`[INFO]`、`[WARN]`、`[ERROR]`）。
3. **讀取並顯示**：使用 `fs.readFile` 讀取 `app.log` 的全部內容，並輸出到 console。
4. **錯誤處理**：若讀取一個不存在的檔案（例如 `missing.log`），正確捕捉並顯示錯誤訊息，而不讓程式崩潰。
5. **進階（選做）**：將上述操作改寫為使用 `fs.promises`（Promise-based API）搭配 `async/await`。

## 範例

```js
const fs = require('fs');
const path = require('path');

const LOG_FILE = path.join(__dirname, 'app.log');

function formatLog(level, message) {
  const timestamp = new Date().toISOString();
  return `[${timestamp}] [${level}] ${message}\n`;
}

// 步驟 1：初始化（覆蓋寫入）
fs.writeFile(LOG_FILE, formatLog('INFO', '伺服器啟動'), (err) => {
  if (err) return console.error('寫入失敗：', err.message);

  // 步驟 2：附加三筆記錄
  const entries = [
    formatLog('INFO',  '收到請求 GET /api/users'),
    formatLog('WARN',  '回應時間超過 500ms'),
    formatLog('ERROR', '資料庫連線逾時'),
  ];

  fs.appendFile(LOG_FILE, entries.join(''), (err) => {
    if (err) return console.error('附加失敗：', err.message);

    // 步驟 3：讀取並顯示
    fs.readFile(LOG_FILE, 'utf8', (err, data) => {
      if (err) return console.error('讀取失敗：', err.message);
      console.log('=== app.log 內容 ===');
      console.log(data);
    });
  });
});

// 步驟 4：讀取不存在的檔案（錯誤處理）
fs.readFile('missing.log', 'utf8', (err, data) => {
  if (err) {
    console.error(`[錯誤處理] 找不到檔案：${err.code} - ${err.message}`);
    return;
  }
  console.log(data);
});
```

預期輸出（大致如下）：

```
[錯誤處理] 找不到檔案：ENOENT - no such file or directory, open 'missing.log'
=== app.log 內容 ===
[2026-07-19T...] [INFO] 伺服器啟動
[2026-07-19T...] [INFO] 收到請求 GET /api/users
[2026-07-19T...] [WARN] 回應時間超過 500ms
[2026-07-19T...] [ERROR] 資料庫連線逾時
```

## 提示
- Callback 是非同步的，注意巢狀順序——若要依序執行，下一步需放在前一步的 callback 內。
- `path.join(__dirname, 'app.log')` 可確保路徑在任何作業系統上都正確。
- `fs.existsSync` 可在操作前先確認檔案是否存在（同步，適合啟動時使用）。
- 錯誤物件的 `err.code` 屬性（如 `ENOENT`、`EACCES`）能幫助你判斷錯誤類型。

## 學習重點（跟後端 API 的關聯）

日誌記錄是每個生產環境 API 的必要功能。Express 中間件（如 Morgan）底層也使用 `fs.appendFile` 或 stream 寫入存取日誌。理解 `fs` 的非同步特性，能幫助你日後正確使用 `async/await` 避免 callback hell，並在 API 路由中安全地讀寫設定檔或上傳的檔案。

## 進階挑戰（選做）

1. **Promise 改寫**：使用 `fs.promises.writeFile` / `appendFile` / `readFile` 搭配 `async/await` 重寫整個流程，讓程式碼更線性易讀。
2. **日誌輪替**：若 `app.log` 超過 1 KB，自動將舊內容備份為 `app.log.bak`，再重新建立空的 `app.log`（提示：先用 `fs.statSync` 取得檔案大小）。
3. **讀取特定行**：讀取 `app.log` 後，只印出包含 `[ERROR]` 的行（提示：`data.split('\n').filter(...)`）。
