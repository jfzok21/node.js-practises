# 第 1 週今日 Node.js 練習題：打造你的第一個 CLI 工具

## 題目說明

在後端開發中，程式不只是「執行就結束」，它需要能夠接受外部設定——例如從命令列傳入參數、從環境變數讀取敏感設定（如資料庫密碼、Port 號碼）。Node.js 提供了 `process.argv`、`process.env` 和 `__dirname` 這三個全域變數，讓你的程式能彈性地因應不同執行環境。本題將帶你用這些工具打造一個實用的 CLI（命令列介面）計算器，並練習透過 `npm scripts` 簡化執行指令。

---

## 技術要求

- **概念：** `process.argv`、`process.env`、`__dirname`、`__filename`、`npm scripts`
- **需要的套件：** 無（Node.js 內建）

---

## 具體任務

1. **探索 `process.argv`**  
   建立 `showArgs.js`，將 `process.argv` 的內容完整印出，接著執行：
   ```bash
   node showArgs.js hello world 123
   ```
   觀察陣列的第 0、1、2 個元素分別是什麼。

2. **建立 CLI 計算器 `calc.js`**  
   從 `process.argv` 讀取三個參數：`數字A`、`運算符號`、`數字B`，執行對應計算並輸出結果。
   - 支援：`+`、`-`、`*`、`/`
   - 若除數為 0，輸出錯誤訊息並以 `process.exit(1)` 結束程式
   - 若參數不足或運算符號無效，輸出使用說明並結束

3. **加入環境變數設定 `config.js`**  
   建立 `config.js`，從 `process.env` 讀取以下變數，並提供預設值：
   - `APP_NAME`（預設：`"MyApp"`）
   - `PORT`（預設：`3000`，轉換為數字）
   - `DEBUG`（預設：`false`，轉換為布林值）
   
   匯出這個設定物件，讓 `calc.js` 在執行時印出目前的設定。

4. **利用 `__dirname` 建立路徑**  
   在 `calc.js` 中，使用 `path.join(__dirname, 'logs')` 印出「log 目錄會在哪裡」，說明為何用 `__dirname` 比相對路徑更安全。

5. **在 `package.json` 的 `scripts` 區塊加入：**
   ```json
   "calc": "node calc.js"
   ```
   讓使用者可以用 `npm run calc -- 10 + 5` 執行。

## 範例

```js
// showArgs.js
console.log(process.argv);
// 執行：node showArgs.js hello 123
// 輸出：
// [ '/usr/bin/node', '/path/to/showArgs.js', 'hello', '123' ]
//   index 0 = Node 執行檔路徑
//   index 1 = 腳本路徑
//   index 2+ = 你傳入的參數

// config.js
module.exports = {
  appName: process.env.APP_NAME || 'MyApp',
  port: Number(process.env.PORT) || 3000,
  debug: process.env.DEBUG === 'true',
};

// calc.js
const path = require('path');
const config = require('./config');

const args = process.argv.slice(2); // 去掉前兩個系統參數

if (args.length < 3) {
  console.log('使用方式：node calc.js <數字A> <運算符> <數字B>');
  console.log('範例：node calc.js 10 + 5');
  process.exit(1);
}

const [a, op, b] = [Number(args[0]), args[1], Number(args[2])];
let result;

switch (op) {
  case '+': result = a + b; break;
  case '-': result = a - b; break;
  case '*': result = a * b; break;
  case '/':
    if (b === 0) {
      console.error('錯誤：除數不能為零');
      process.exit(1);
    }
    result = a / b;
    break;
  default:
    console.error(`不支援的運算符：${op}`);
    process.exit(1);
}

console.log(`[${config.appName}] ${a} ${op} ${b} = ${result}`);
if (config.debug) {
  console.log(`DEBUG: __dirname = ${__dirname}`);
  console.log(`DEBUG: log 目錄路徑 = ${path.join(__dirname, 'logs')}`);
}
```

**預期執行結果：**
```bash
$ node calc.js 10 + 5
[MyApp] 10 + 5 = 15

$ APP_NAME=計算器 DEBUG=true node calc.js 20 / 4
[計算器] 20 / 4 = 5
DEBUG: __dirname = /home/user/my-project
DEBUG: log 目錄路徑 = /home/user/my-project/logs

$ node calc.js 10 / 0
錯誤：除數不能為零
```

## 提示

- `process.argv` 是一個**字串陣列**，記得用 `Number()` 將數字參數轉型
- `process.argv.slice(2)` 是標準寫法，用來略去 Node 路徑和腳本路徑
- `process.exit(0)` 表示正常結束；`process.exit(1)` 表示錯誤結束（CI/CD 系統會依此判斷是否成功）
- `__dirname` 是執行時當前腳本的**絕對目錄路徑**，不受執行位置影響；相對路徑 `./` 則以「終端機所在目錄」為準，容易出錯

---

## 學習重點（跟後端 API 的關聯）

在真實的 Express 後端中：

| 今天練習 | 對應的後端用途 |
|---|---|
| `process.env.PORT` | 讓伺服器的 Port 可由部署平台（Heroku、Render）動態設定 |
| `process.env.DB_URL` | 資料庫連線字串，絕對不能寫死在程式碼中 |
| `process.exit(1)` | 啟動前的健康檢查失敗時，強制停止服務 |
| `__dirname` | 靜態檔案路徑、multer 上傳目錄，確保路徑正確 |

你日後看到的 `dotenv` 套件，本質上就是把 `.env` 檔案的內容注入 `process.env`——它的基礎就是今天練習的這個機制。
