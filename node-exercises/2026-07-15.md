# 第 2 週今日 Node.js 練習題：用 fs 模組打造簡易日誌系統

## 題目說明

建立一個命令列工具，能夠將訊息以時間戳記的格式寫入日誌檔案，並支援讀取、清空等基本操作。透過這個練習，你將掌握 Node.js 內建 `fs` 模組的核心用法。

---

## 技術要求

- **概念**：`fs.readFileSync`、`fs.writeFileSync`、`fs.appendFileSync`、`fs.existsSync`、同步 vs 非同步 API
- **需要的套件**：無（僅使用 Node.js 內建模組）

---

## 具體任務

1. 建立 `logger.js`，匯出下列三個函式：
   - `log(message)` — 將訊息附加到 `app.log`，格式為 `[YYYY-MM-DD HH:mm:ss] message`
   - `readLog()` — 讀取並回傳 `app.log` 的完整內容（若檔案不存在則回傳空字串）
   - `clearLog()` — 清空 `app.log`（將內容覆寫為空字串）

2. 建立 `index.js`，依照命令列引數呼叫對應功能：
   - `node index.js write "今天完成了第一個練習"` → 寫入一筆日誌
   - `node index.js read` → 顯示所有日誌
   - `node index.js clear` → 清空日誌

3. 每次寫入後，`read` 指令要能正確顯示新增的內容。

---

## 範例

```js
// logger.js
const fs = require('fs');
const path = require('path');

const LOG_FILE = path.join(__dirname, 'app.log');

function log(message) {
  const timestamp = new Date().toISOString().replace('T', ' ').substring(0, 19);
  const line = `[${timestamp}] ${message}\n`;
  fs.appendFileSync(LOG_FILE, line, 'utf8');
}

function readLog() {
  if (!fs.existsSync(LOG_FILE)) return '';
  return fs.readFileSync(LOG_FILE, 'utf8');
}

function clearLog() {
  fs.writeFileSync(LOG_FILE, '', 'utf8');
}

module.exports = { log, readLog, clearLog };
```

```js
// index.js
const { log, readLog, clearLog } = require('./logger');

const [, , command, ...rest] = process.argv;

if (command === 'write') {
  const message = rest.join(' ');
  log(message);
  console.log('✅ 已寫入日誌');
} else if (command === 'read') {
  const content = readLog();
  console.log(content || '（日誌為空）');
} else if (command === 'clear') {
  clearLog();
  console.log('🗑️  日誌已清空');
} else {
  console.log('用法：node index.js [write|read|clear] [訊息]');
}
```

執行結果：

```
$ node index.js write "完成了 fs 模組練習"
✅ 已寫入日誌

$ node index.js write "今天學會了 appendFileSync"
✅ 已寫入日誌

$ node index.js read
[2026-07-15 10:30:00] 完成了 fs 模組練習
[2026-07-15 10:30:05] 今天學會了 appendFileSync

$ node index.js clear
🗑️  日誌已清空

$ node index.js read
（日誌為空）
```

---

## 提示

- `path.join(__dirname, 'app.log')` 確保無論從哪個目錄執行，都能找到正確的檔案路徑。
- `fs.appendFileSync` 不會覆蓋原有內容，而是在末尾附加，適合日誌寫入。
- `fs.writeFileSync` 搭配空字串 `''` 可以清空檔案而不刪除它。
- `new Date().toISOString()` 格式為 `2026-07-15T10:30:00.000Z`，可以用 `.replace` 和 `.substring` 整理成易讀格式。

---

## 學習重點（跟後端 API 的關聯）

日誌系統是後端 API 的基礎元件之一：

| 本練習技能 | 後端 API 應用場景 |
|---|---|
| `appendFileSync` 寫入日誌 | API Server 記錄每個 request/response |
| `existsSync` 檢查檔案 | 初始化時確認設定檔是否存在 |
| `readFileSync` 讀取設定 | 啟動時載入 `.env` 或 JSON 設定檔 |
| `module.exports` 匯出模組 | 將 logger 抽成共用工具，供多個路由引用 |

在實際的 Express API 中，你會使用 `morgan` 或 `winston` 這類基於 `fs` 的套件來處理日誌，原理與本練習完全相同。

## 進階挑戰（選做）

1. **非同步版本**：將 `log`、`readLog`、`clearLog` 改為使用 `fs.promises`（`async/await`），在 `index.js` 中用 `async` IIFE 呼叫。

2. **日誌等級**：支援 `log(message, level)` 參數，等級有 `INFO`、`WARN`、`ERROR`，輸出格式改為 `[2026-07-15 10:30:00] [ERROR] 發生例外`。

3. **依日期分檔**：每天自動建立一個新的日誌檔（如 `app-2026-07-15.log`），避免單一檔案無限成長。
