# 第 1 週今日 Node.js 練習題：打造你的第一個 CLI 工具

## 題目說明

前兩題練習了模組拆分（`require` / `module.exports`）。今天要把這個基礎延伸到「互動式命令列工具」：透過 `process.argv` 讀取使用者在終端機輸入的參數，搭配模組化設計，打造一個可以從命令列呼叫的轉換工具。這個模式在真實後端專案中很常見——例如資料庫 seed 腳本、一次性資料轉換工具、或 CI/CD 自動化指令，都是以 CLI 腳本形式執行的。

---

## 技術要求

- **概念：** `process.argv`、`process.env`、`module.exports`、`require`、npm scripts
- **需要的套件：** 無（Node.js 內建）

---

## 具體任務

1. 建立 `converters.js`，用 `module.exports` 匯出以下轉換函式：
   - `celsiusToFahrenheit(c)` → 攝氏轉華氏（公式：`c * 9/5 + 32`）
   - `kmToMiles(km)` → 公里轉英里（公式：`km * 0.621371`）
   - `twdToUsd(twd, rate)` → 台幣轉美元（`twd / rate`）

2. 建立 `cli.js`，作為命令列入口：
   - 從 `process.argv` 讀取第三、四個參數作為「指令」和「數值」
   - 根據指令呼叫對應的 `converters.js` 函式，印出結果
   - 若指令不存在，印出使用說明

3. 在 `package.json` 的 `scripts` 欄位加入三個快捷指令：
   ```json
   "c2f": "node cli.js c2f",
   "km2mi": "node cli.js km2mi",
   "twd2usd": "node cli.js twd2usd"
   ```

4. 執行以下指令驗證結果：
   ```
   node cli.js c2f 100
   node cli.js km2mi 10
   node cli.js twd2usd 1000
   node cli.js unknown
   ```

---

## 範例

```js
// converters.js
module.exports = {
  celsiusToFahrenheit: (c) => c * 9 / 5 + 32,
  kmToMiles: (km) => km * 0.621371,
  twdToUsd: (twd, rate = 32) => twd / rate,
};

// cli.js
const converters = require('./converters');

const command = process.argv[2];
const value = parseFloat(process.argv[3]);

switch (command) {
  case 'c2f':
    console.log(`${value}°C = ${converters.celsiusToFahrenheit(value).toFixed(2)}°F`);
    break;
  case 'km2mi':
    console.log(`${value} 公里 = ${converters.kmToMiles(value).toFixed(4)} 英里`);
    break;
  case 'twd2usd':
    console.log(`NT$${value} ≈ $${converters.twdToUsd(value).toFixed(2)} USD（匯率 32）`);
    break;
  default:
    console.log('使用方式：');
    console.log('  node cli.js c2f   <攝氏>    - 攝氏轉華氏');
    console.log('  node cli.js km2mi <公里>    - 公里轉英里');
    console.log('  node cli.js twd2usd <台幣>  - 台幣轉美元');
}
```

**預期輸出：**
```
$ node cli.js c2f 100
100°C = 212.00°F

$ node cli.js km2mi 10
10 公里 = 6.2137 英里

$ node cli.js twd2usd 1000
NT$1000 ≈ $31.25 USD（匯率 32）

$ node cli.js unknown
使用方式：
  node cli.js c2f   <攝氏>    - 攝氏轉華氏
  node cli.js km2mi <公里>    - 公里轉英里
  node cli.js twd2usd <台幣>  - 台幣轉美元
```

---

## 提示

- `process.argv` 是一個陣列：`[0]` 是 node 執行檔路徑，`[1]` 是腳本路徑，**`[2]` 才是你的第一個參數**。
- `parseFloat(undefined)` 會回傳 `NaN`，可加 `if (isNaN(value))` 做防呆。
- npm script 中的 `node cli.js c2f` 執行後，`npm run c2f 100` 的 `100` 會自動附在後面（npm 7+ 需加 `--` 分隔：`npm run c2f -- 100`）。

## 學習重點（跟後端 API 的關聯）

後端專案中常見的 CLI 腳本應用：
- **資料庫 seed：** `node scripts/seed.js` 插入測試資料
- **資料遷移：** `node scripts/migrate.js up` / `down`
- **部署前處理：** `node scripts/build-config.js production`

這些腳本的結構與今天完全一樣：`process.argv` 讀取指令 → `require` 載入模組 → 執行邏輯。掌握後，將來寫 `npm run seed` 或 `npm run migrate` 就會毫無壓力。

## 進階挑戰（選做）

1. 讀取 `process.env.EXCHANGE_RATE`：若環境變數存在，用它取代預設匯率 32。  
   執行方式：`EXCHANGE_RATE=31.5 node cli.js twd2usd 1000`

2. 加入 `--help` 旗標支援，讓 `node cli.js --help` 也能印出使用說明。

3. 若使用者未提供數值（`process.argv[3]` 為 `undefined`），印出友善錯誤訊息並以非零狀態碼退出（`process.exit(1)`）。
