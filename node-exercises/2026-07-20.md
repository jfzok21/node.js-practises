# 第 2 週今日 Node.js 練習題：檔案讀寫與目錄操作

## 題目說明

使用 Node.js 內建的 `fs` 模組，建立一個簡單的「筆記管理工具」。此工具能夠讀取、寫入、列出與刪除文字筆記，所有筆記以 `.txt` 檔案形式儲存在本地目錄中。

---

## 技術要求

- **概念：** 同步 vs 非同步 fs 操作、路徑處理（`path` 模組）、錯誤處理
- **需要的套件：** 僅使用 Node.js 內建模組（`fs`、`path`）

---

## 具體任務

1. **建立 `notes/` 目錄**
   - 使用 `fs.mkdirSync` 或 `fs.mkdir` 確保 `notes/` 目錄存在（若已存在不報錯）。

2. **新增筆記（`addNote`）**
   - 接收 `title`（檔名）與 `content`（內容）兩個參數。
   - 將內容寫入 `notes/<title>.txt`，若同名檔案已存在則覆蓋。

3. **讀取筆記（`getNote`）**
   - 接收 `title`，讀取對應檔案內容並回傳字串。
   - 若檔案不存在，回傳 `null` 或印出錯誤訊息。

4. **列出所有筆記（`listNotes`）**
   - 讀取 `notes/` 目錄下所有 `.txt` 檔案名稱，印出筆記清單（去除副檔名）。

5. **刪除筆記（`deleteNote`）**
   - 接收 `title`，刪除對應檔案。
   - 若檔案不存在，給出友善提示而非直接拋出錯誤。

---

## 範例

```js
// notes-manager.js
const fs = require('fs');
const path = require('path');

const NOTES_DIR = path.join(__dirname, 'notes');

// 確保目錄存在
fs.mkdirSync(NOTES_DIR, { recursive: true });

function addNote(title, content) {
  const filePath = path.join(NOTES_DIR, `${title}.txt`);
  fs.writeFileSync(filePath, content, 'utf8');
  console.log(`✅ 筆記「${title}」已儲存`);
}

function getNote(title) {
  const filePath = path.join(NOTES_DIR, `${title}.txt`);
  if (!fs.existsSync(filePath)) {
    console.log(`❌ 找不到筆記「${title}」`);
    return null;
  }
  return fs.readFileSync(filePath, 'utf8');
}

function listNotes() {
  const files = fs.readdirSync(NOTES_DIR).filter(f => f.endsWith('.txt'));
  if (files.length === 0) {
    console.log('（目前沒有任何筆記）');
    return;
  }
  console.log('📝 筆記清單：');
  files.forEach(f => console.log(' -', path.basename(f, '.txt')));
}

function deleteNote(title) {
  const filePath = path.join(NOTES_DIR, `${title}.txt`);
  if (!fs.existsSync(filePath)) {
    console.log(`⚠️ 筆記「${title}」不存在，無法刪除`);
    return;
  }
  fs.unlinkSync(filePath);
  console.log(`🗑️ 筆記「${title}」已刪除`);
}

// 測試流程
addNote('shopping', '牛奶、雞蛋、麵包');
addNote('todo', '完成 Node.js 練習、喝水');
listNotes();
console.log('\n讀取 shopping 筆記：\n' + getNote('shopping'));
deleteNote('todo');
deleteNote('memo'); // 不存在的筆記
listNotes();
```

**預期輸出：**
```
✅ 筆記「shopping」已儲存
✅ 筆記「todo」已儲存
📝 筆記清單：
 - shopping
 - todo
讀取 shopping 筆記：
牛奶、雞蛋、麵包
🗑️ 筆記「todo」已刪除
⚠️ 筆記「memo」不存在，無法刪除
📝 筆記清單：
 - shopping
```

---

## 提示

- `fs.mkdirSync(path, { recursive: true })` 可以安全地建立目錄，即使已存在也不報錯。
- `path.join(__dirname, 'notes')` 使用絕對路徑，避免因執行目錄不同而出錯。
- `fs.existsSync(filePath)` 可在讀取或刪除前先確認檔案是否存在。
- 試著將 `writeFileSync` 改成 `writeFile`（非同步版本），觀察 callback 的使用方式。

---

## 學習重點（跟後端 API 的關聯）

在實際後端開發中，`fs` 模組的應用場景非常廣泛：

- **設定檔讀取：** 伺服器啟動時用 `fs.readFileSync` 載入 `.json` 或 `.env` 設定。
- **Log 寫入：** 將 API 存取日誌寫入檔案，方便除錯與監控。
- **檔案上傳儲存：** 搭配 multer 等套件，將使用者上傳的圖片或文件儲存到伺服器。
- **靜態資源服務：** Express 的 `express.static` 底層也是透過 `fs` 讀取檔案並回應給瀏覽器。

熟悉同步與非同步 fs 操作，是處理 I/O 密集型後端任務的基礎。

## 進階挑戰（選做）

1. **改用非同步版本（Promise）：** 將所有 `fs.xxxSync` 改成 `fs.promises.xxx`，並以 `async/await` 包裝，回傳 Promise。

2. **新增搜尋功能（`searchNotes`）：** 讀取所有筆記內容，找出包含指定關鍵字的筆記，印出其標題與匹配的行。

3. **JSON 格式支援：** 建立 `addJsonNote(title, data)` 與 `getJsonNote(title)`，以 `.json` 格式儲存物件資料（使用 `JSON.stringify` / `JSON.parse`）。

4. **命令列介面：** 讓 `notes-manager.js` 可以透過指令列操作，例如：
   ```bash
   node notes-manager.js add "meeting" "下午三點開會"
   node notes-manager.js list
   node notes-manager.js get "meeting"
   node notes-manager.js delete "meeting"
   ```
   提示：使用 `process.argv` 取得命令列參數。
