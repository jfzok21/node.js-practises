# 第 2 週今日 Node.js 練習題：用 fs 模組建立日誌系統

## 題目說明

撰寫一個簡單的「日誌記錄器」（Logger），能夠將訊息寫入文字檔，並在需要時讀取歷史記錄。這個練習模擬後端 API 中常見的 logging 需求，讓你熟悉 Node.js 內建的 `fs` 模組。

## 技術要求

- 概念：同步與非同步檔案讀寫、路徑處理、錯誤處理
- 需要的套件：無（僅使用 Node.js 內建模組 `fs` 與 `path`）

## 具體任務

1. **寫入日誌**：建立 `writeLog(message)` 函式，將 `[時間戳記] message` 這樣格式的文字附加（append）到 `logs/app.log` 檔案末尾。
2. **讀取日誌**：建立 `readLogs()` 函式，讀取 `logs/app.log` 的全部內容並印出到 console。
3. **確保資料夾存在**：程式啟動時，若 `logs/` 目錄不存在，自動建立它（使用 `fs.mkdirSync` 搭配 `{ recursive: true }` 選項）。
4. **呼叫示範**：在 `index.js` 中依序呼叫 `writeLog`、`writeLog`、`readLogs`，確認兩筆記錄都出現在輸出中。

## 範例

```js
// 輸入（呼叫方式）
writeLog('伺服器已啟動');
writeLog('收到 GET /api/users 請求');
readLogs();

// 預期輸出（logs/app.log 與 console）
// [2026-07-16T08:00:00.000Z] 伺服器已啟動
// [2026-07-16T08:00:01.000Z] 收到 GET /api/users 請求
```

## 提示

- 使用 `fs.appendFileSync(filePath, content, 'utf8')` 做附加寫入；這樣不會覆蓋原有內容。
- 時間戳記可用 `new Date().toISOString()` 取得。
- 使用 `path.join(__dirname, 'logs', 'app.log')` 組合路徑，避免跨平台路徑問題。
- `fs.readFileSync(filePath, 'utf8')` 可直接回傳字串內容。

## 學習重點（跟後端 API 的關聯）

在真實的 Express 或 Fastify 後端中，每一筆 HTTP 請求通常都會被記錄下來，方便之後除錯或監控。`fs` 模組就是最基礎的實作方式——雖然正式專案會改用 `winston` 或 `pino` 這類日誌套件，但這些套件底層都依賴相同的非同步 I/O 概念。理解 `fs` 的運作，有助於你之後看懂這些工具的原始碼與設定。

## 進階挑戰（選做）

- 將 `writeLog` 改成非同步版本，使用 `fs.promises.appendFile`（回傳 Promise），並在 `index.js` 以 `async/await` 呼叫。
- 為 `readLogs` 加上行數計數，讓每行輸出前面顯示行號（例如 `1: [時間] 訊息`）。
- 加入 `clearLogs()` 函式，用 `fs.writeFileSync` 清空日誌檔案（寫入空字串）。
