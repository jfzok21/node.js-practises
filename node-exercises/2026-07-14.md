# 第 2 週今日 Node.js 練習題：讀寫與操作本地檔案系統

## 題目說明

學習使用 Node.js 內建的 `fs`（File System）模組，對本地檔案進行讀取、寫入、附加內容與刪除操作，並利用非同步 API 避免阻塞主執行緒。

---

## 技術要求

- **概念**：`fs.readFile`、`fs.writeFile`、`fs.appendFile`、`fs.unlink`、`fs.existsSync`、路徑處理（`path` 模組）
- **需要的套件**：無（全部使用 Node.js 內建模組）

---

## 具體任務

1. **建立一個 `notes.js` 腳本**，要求能執行以下四個子命令（透過 `process.argv` 傳入）：
   - `add <內容>` — 將文字附加一行到 `notes.txt`
   - `read` — 讀取並印出 `notes.txt` 的所有內容
   - `clear` — 清空（覆蓋為空字串）`notes.txt`
   - `delete` — 若 `notes.txt` 存在則刪除，否則印出「找不到檔案」

2. 每次執行 `add` 時，在內容後面自動補上換行符號 `\n`，並在行首加上時間戳記，格式為 `[HH:MM:SS]`。

3. 所有 fs 操作必須使用 **async/await**（搭配 `fs.promises` API），並用 `try/catch` 處理錯誤。

4. 使用 `path.join(__dirname, 'notes.txt')` 組合檔案路徑，確保跨平台相容。

---

## 範例

```js
// 執行：node notes.js add "今天學了 fs 模組"
// 輸出：✅ 已新增筆記

// 執行：node notes.js read
// 輸出：
// [09:30:15] 今天學了 fs 模組

// 執行：node notes.js delete
// 輸出：🗑️ 已刪除 notes.txt

// 執行：node notes.js delete  （再執行一次）
// 輸出：⚠️ 找不到檔案：notes.txt
```

---

## 提示

- `fs.promises` 是 `require('fs').promises` 或 `require('fs/promises')`，兩者等價。
- `fs.existsSync()` 是同步方法，可在刪除前快速檢查檔案是否存在，但主要操作仍建議用非同步 API。
- `process.argv[2]` 取得子命令，`process.argv[3]` 取得第一個參數。
- 取得當前時間字串可用：
  ```js
  const now = new Date();
  const timestamp = now.toTimeString().slice(0, 8); // "HH:MM:SS"
  ```
- 附加內容用 `fs.promises.appendFile(path, content)`，不會覆蓋原本資料。

## 學習重點

**與後端 API 的關聯：**

後端伺服器常需要操作檔案系統，例如：
- 讀取設定檔（`.env`、`config.json`）以初始化應用程式
- 將上傳檔案存到本地磁碟（第 11 週 multer 的底層即是此概念）
- 寫入應用程式日誌（log files）以供除錯
- 在無資料庫的小型工具中，用 JSON 檔作為輕量級資料儲存

掌握 `fs` 模組是理解後續中介層（middleware）和檔案上傳功能的基礎。

## 進階挑戰（選做）

1. **JSON 資料儲存**：將每筆筆記儲存為 `notes.json`（陣列格式），每筆包含 `{ id, timestamp, content }`，並實作 `node notes.js remove <id>` 根據 id 刪除單筆筆記。

2. **目錄備份**：新增 `backup` 子命令，將 `notes.txt` 複製為 `notes_backup_[YYYY-MM-DD].txt`（使用 `fs.promises.copyFile`）。

3. **監聽檔案變更**：使用 `fs.watch('notes.txt', callback)` 在每次檔案被更動時，自動在終端機印出「📝 notes.txt 已更新」。
