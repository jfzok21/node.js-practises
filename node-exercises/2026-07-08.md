# 第 1 週今日 Node.js 練習題：打造你的第一個模組系統

## 題目說明

你將建立一個簡單的「計算機模組」，學習如何用 `module.exports` 匯出功能、用 `require` 引入模組，以及如何初始化一個 npm 專案。這是後端開發的最基礎磚塊：將程式碼分成可重用的模組。

---

## 技術要求

- **概念：** `require`、`module.exports`、npm init、Node.js 模組系統（CommonJS）
- **需要的套件：** 無（使用 Node.js 內建功能即可）

---

## 具體任務

1. 在專案根目錄執行 `npm init -y`，初始化一個 npm 專案，確認 `package.json` 被正確建立。

2. 建立檔案 `calculator.js`，用 `module.exports` 匯出以下四個函式：
   - `add(a, b)` — 回傳 a + b
   - `subtract(a, b)` — 回傳 a - b
   - `multiply(a, b)` — 回傳 a * b
   - `divide(a, b)` — 回傳 a / b（若 b 為 0，需回傳字串 `"Error: 除數不能為零"`）

3. 建立檔案 `index.js`，用 `require` 引入 `calculator.js`，並印出以下結果：
   - `10 + 3 = ?`
   - `10 - 3 = ?`
   - `10 * 3 = ?`
   - `10 / 3 = ?`（四捨五入至小數點第二位）
   - `10 / 0 = ?`（驗證錯誤處理）

4. 用 `node index.js` 執行並確認輸出正確。

---

## 範例

```js
// calculator.js
module.exports = {
  add: (a, b) => a + b,
  subtract: (a, b) => a - b,
  multiply: (a, b) => a * b,
  divide: (a, b) => {
    if (b === 0) return "Error: 除數不能為零";
    return a / b;
  },
};

// index.js
const calc = require("./calculator");

console.log(`10 + 3 = ${calc.add(10, 3)}`);         // 10 + 3 = 13
console.log(`10 - 3 = ${calc.subtract(10, 3)}`);    // 10 - 3 = 7
console.log(`10 * 3 = ${calc.multiply(10, 3)}`);    // 10 * 3 = 30
console.log(`10 / 3 = ${calc.divide(10, 3).toFixed(2)}`); // 10 / 3 = 3.33
console.log(`10 / 0 = ${calc.divide(10, 0)}`);      // 10 / 0 = Error: 除數不能為零
```

**預期輸出：**
```
10 + 3 = 13
10 - 3 = 7
10 * 3 = 30
10 / 3 = 3.33
10 / 0 = Error: 除數不能為零
```

---

## 提示

- `module.exports` 可以匯出一個物件，包含多個函式。
- `require('./calculator')` 中的 `./` 代表「同一目錄下」，不能省略。
- `npm init -y` 的 `-y` 旗標代表全部使用預設值，快速生成 `package.json`。
- `toFixed(2)` 是 JavaScript Number 的方法，會回傳**字串**，所以 `divide` 回傳的若已是字串（錯誤情況），不要再呼叫 `.toFixed()`。

---

## 學習重點（跟後端 API 的關聯）

在真實的 Express API 專案中，你會把路由（routes）、資料庫邏輯（services）、工具函式（utils）分成獨立的檔案，再用 `require` 或 ES Module 的 `import` 把它們組裝在一起。今天練習的模組化思維，正是整個後端架構的基礎：

- `calculator.js` → 類比未來的 `userService.js`（商業邏輯層）
- `index.js` → 類比未來的 `app.js`（應用程式進入點）

把邏輯分離成模組，讓程式碼可測試、可維護、可重用。

## 進階挑戰（選做）

1. **具名匯出練習：** 嘗試將 `calculator.js` 改成分別 `module.exports.add = ...` 的形式，而不是匯出整個物件，確認 `index.js` 仍能正常運作。

2. **加入 `power(base, exponent)` 函式：** 計算次方（例如 `power(2, 10)` 回傳 `1024`），並驗證 `power(2, 0)` 回傳 `1`。

3. **建立第二個模組 `formatter.js`：** 匯出一個 `formatResult(expression, result)` 函式，讓 `index.js` 的輸出格式統一由這個模組負責——練習多模組協作。
