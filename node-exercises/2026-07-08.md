# 第 1 週今日 Node.js 練習題：打造你的第一個模組系統

## 題目說明

你將建立一個「個人資料計算工具」，學習如何將不同功能拆成獨立檔案，透過 `module.exports` 匯出、`require` 引入，並用 `npm init` 初始化專案。這是所有 Node.js 後端開發的根基。

---

## 技術要求

- **概念：** CommonJS 模組系統、`require`、`module.exports`、`npm init`
- **需要的套件：** 無（Node.js 內建）

---

## 具體任務

1. 執行 `npm init -y` 初始化專案，確認 `package.json` 產生。

2. 建立 `mathUtils.js`，用 `module.exports` 匯出以下函式：
   - `add(a, b)` → 回傳 a + b
   - `subtract(a, b)` → 回傳 a - b
   - `multiply(a, b)` → 回傳 a * b
   - `divide(a, b)` → 回傳 a / b；若 b 為 0，回傳字串 `"錯誤：除數不能為零"`

3. 建立 `stringUtils.js`，匯出以下函式：
   - `toUpperCase(str)` → 全部轉大寫
   - `countWords(str)` → 回傳字串中的單字數量（以空白分隔）
   - `reverse(str)` → 回傳反轉後的字串

4. 建立 `index.js`，同時引入上述兩個模組，並印出：
   - `15 + 7`、`15 - 7`、`15 * 7`、`15 / 7`（小數點兩位）、`15 / 0`
   - `"hello world"` 轉大寫、單字數、反轉結果

5. 執行 `node index.js` 驗證輸出。

---

## 範例

```js
// mathUtils.js
module.exports = {
  add: (a, b) => a + b,
  subtract: (a, b) => a - b,
  multiply: (a, b) => a * b,
  divide: (a, b) => (b === 0 ? "錯誤：除數不能為零" : a / b),
};

// stringUtils.js
module.exports = {
  toUpperCase: (str) => str.toUpperCase(),
  countWords: (str) => str.trim().split(/\s+/).length,
  reverse: (str) => str.split("").reverse().join(""),
};

// index.js
const math = require("./mathUtils");
const str = require("./stringUtils");

console.log(`15 + 7 = ${math.add(15, 7)}`);
console.log(`15 - 7 = ${math.subtract(15, 7)}`);
console.log(`15 * 7 = ${math.multiply(15, 7)}`);
console.log(`15 / 7 = ${math.divide(15, 7).toFixed(2)}`);
console.log(`15 / 0 = ${math.divide(15, 0)}`);

console.log(`大寫：${str.toUpperCase("hello world")}`);
console.log(`單字數：${str.countWords("hello world")}`);
console.log(`反轉：${str.reverse("hello world")}`);
```

**預期輸出：**
```
15 + 7 = 22
15 - 7 = 8
15 * 7 = 105
15 / 7 = 2.14
15 / 0 = 錯誤：除數不能為零
大寫：HELLO WORLD
單字數：2
反轉：dlrow olleh
```

---

## 提示

- `require('./mathUtils')` 的 `./` 不能省略，否則 Node 會以為你在引入 npm 套件。
- `module.exports` 可以是物件（多個函式），也可以是單一函式或值。
- `str.trim().split(/\s+/)` 使用正則表達式，能正確處理多個連續空白。

---

## 學習重點（跟後端 API 的關聯）

真實的 Express 專案會將程式碼分層：

```
src/
├── routes/userRoutes.js      ← 路由（類比今天的 index.js）
├── services/userService.js   ← 商業邏輯（類比今天的 mathUtils.js）
└── utils/formatHelper.js     ← 工具函式（類比今天的 stringUtils.js）
```

今天練習的「拆模組 → require 組合」正是這個分層架構的縮影。
