# 第 1 週今日 Node.js 練習題：事件驅動程式設計入門（EventEmitter）

## 題目說明

Node.js 的核心概念之一是「事件驅動」架構（Event-Driven Architecture）。幾乎所有 Node.js 的內建模組——HTTP 伺服器、檔案系統串流、網路連線——底層都是以 `EventEmitter` 為基礎實作的。今天你將直接使用 Node.js 內建的 `events` 模組，學習如何「發佈事件」與「訂閱事件」，建立一個簡單的訂單處理系統，體驗事件驅動的設計思維。

---

## 技術要求

- **概念：** `events` 模組、`EventEmitter`、`emit`、`on`、`once`、繼承 EventEmitter
- **需要的套件：** 無（Node.js 內建）

---

## 具體任務

1. 用 `require('events')` 引入 `EventEmitter`，建立一個 `orderEmitter` 實例。

2. 使用 `.on()` 訂閱以下三個事件：
   - `'order:placed'`：印出 `[訂單成立] 訂單編號: #xxx，商品: yyy，金額: $zzz`
   - `'order:shipped'`：印出 `[出貨通知] 訂單 #xxx 已於 [時間] 出貨`
   - `'order:cancelled'`：印出 `[訂單取消] 訂單 #xxx 已取消，原因: yyy`

3. 使用 `.once()` 訂閱 `'app:ready'` 事件：只在第一次觸發時印出 `[系統啟動] 訂單系統已就緒`，之後再次觸發不做任何事。

4. 建立一個繼承自 `EventEmitter` 的 `OrderSystem` 類別，加入以下方法：
   - `placeOrder(id, product, price)`：觸發 `'order:placed'` 事件
   - `shipOrder(id)`：觸發 `'order:shipped'` 事件（包含當前時間）
   - `cancelOrder(id, reason)`：觸發 `'order:cancelled'` 事件

5. 建立 `OrderSystem` 實例，依序執行以下操作並觀察輸出：
   - 觸發 `'app:ready'` 兩次（驗證 `once` 只觸發一次）
   - 建立三筆訂單
   - 出貨其中兩筆
   - 取消其中一筆

## 範例

```js
const EventEmitter = require('events');

// 步驟 2-3：直接使用 emitter 實例
const orderEmitter = new EventEmitter();

orderEmitter.once('app:ready', () => {
  console.log('[系統啟動] 訂單系統已就緒');
});

orderEmitter.on('order:placed', ({ id, product, price }) => {
  console.log(`[訂單成立] 訂單編號: #${id}，商品: ${product}，金額: $${price}`);
});

orderEmitter.on('order:shipped', ({ id, time }) => {
  console.log(`[出貨通知] 訂單 #${id} 已於 ${time} 出貨`);
});

orderEmitter.on('order:cancelled', ({ id, reason }) => {
  console.log(`[訂單取消] 訂單 #${id} 已取消，原因: ${reason}`);
});

// 步驟 4：繼承 EventEmitter 的類別
class OrderSystem extends EventEmitter {
  placeOrder(id, product, price) {
    this.emit('order:placed', { id, product, price });
  }

  shipOrder(id) {
    const time = new Date().toLocaleTimeString('zh-TW');
    this.emit('order:shipped', { id, time });
  }

  cancelOrder(id, reason) {
    this.emit('order:cancelled', { id, reason });
  }
}

// 步驟 5：測試
const system = new OrderSystem();

// 訂閱事件（綁定在 system 上）
system.once('app:ready', () => console.log('[系統啟動] 訂單系統已就緒'));
system.on('order:placed', ({ id, product, price }) =>
  console.log(`[訂單成立] 訂單編號: #${id}，商品: ${product}，金額: $${price}`)
);
system.on('order:shipped', ({ id, time }) =>
  console.log(`[出貨通知] 訂單 #${id} 已於 ${time} 出貨`)
);
system.on('order:cancelled', ({ id, reason }) =>
  console.log(`[訂單取消] 訂單 #${id} 已取消，原因: ${reason}`)
);

system.emit('app:ready');   // 印出啟動訊息
system.emit('app:ready');   // 不做任何事（once 已用掉）

system.placeOrder('A001', 'MacBook Pro', 65000);
system.placeOrder('A002', 'AirPods Pro', 7490);
system.placeOrder('A003', 'iPad Air', 19900);

system.shipOrder('A001');
system.shipOrder('A002');

system.cancelOrder('A003', '缺貨');
```

**預期輸出：**
```
[系統啟動] 訂單系統已就緒
[訂單成立] 訂單編號: #A001，商品: MacBook Pro，金額: $65000
[訂單成立] 訂單編號: #A002，商品: AirPods Pro，金額: $7490
[訂單成立] 訂單編號: #A003，商品: iPad Air，金額: $19900
[出貨通知] 訂單 #A001 已於 xx:xx:xx 出貨
[出貨通知] 訂單 #A002 已於 xx:xx:xx 出貨
[訂單取消] 訂單 #A003 已取消，原因: 缺貨
```

## 提示

- `emitter.on(event, listener)` 每次事件觸發都會執行；`emitter.once(event, listener)` 只執行一次後自動移除。
- `emit` 的第二個以後的參數會依序傳給 listener，也可以傳一個物件（如 `{ id, product }`）更清晰。
- `class OrderSystem extends EventEmitter` 讓你的類別擁有所有 EventEmitter 的方法，無需額外設定。
- 用 `emitter.listenerCount('order:placed')` 可以查看某事件目前有幾個監聽器，方便除錯。
- 若要移除特定監聽器，使用 `emitter.off(event, listener)`（需先把 listener 存成具名函式）。

## 學習重點（跟後端 API 的關聯）

Node.js 的 HTTP 模組本質上就是 EventEmitter：

```js
const http = require('http');
const server = http.createServer();

// 底層等同於：
server.on('request', (req, res) => { ... });  // 每次收到請求時觸發
server.on('error', (err) => { ... });         // 發生錯誤時觸發
```

Express 的 `app`、資料庫連線池（如 `pg` 的 Pool）、以及串流（Stream）都繼承自 EventEmitter。理解今天的 `on` / `emit` / `once`，就能看懂這些框架的底層行為，遇到事件相關的 bug 也能更快定位。
