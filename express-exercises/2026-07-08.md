# 第 1 週今日練習題：Express 基礎路由 — 待辦事項 API

## 題目說明

使用 Express.js 建立一個簡單的「待辦事項（Todo）」REST API，以純記憶體陣列儲存資料（不需接資料庫），實作 GET 與 POST 兩種 HTTP 方法，並以 JSON 格式回應。這道題目讓你熟悉 Express 伺服器的建立、路由的設定以及 JSON 資料的傳入與傳出。

---

## 技術要求

- **Express.js**：建立 HTTP 伺服器、定義路由（`app.get`、`app.post`）、解析 JSON 請求體（`express.json()`）
- **PostgreSQL**：本週不使用，資料以 JS 陣列暫存即可

---

## 具體任務

1. **初始化專案**
   - 在 `week1/` 資料夾下執行 `npm init -y`，安裝 `express`
   - 建立 `index.js` 作為主程式進入點

2. **啟動 Express 伺服器**
   - 監聽埠號 `3000`
   - 伺服器啟動後印出 `Server running on http://localhost:3000`

3. **GET `/todos`**
   - 回傳所有待辦事項清單，格式：
     ```json
     {
       "todos": [
         { "id": 1, "title": "買早餐", "done": false }
       ]
     }
     ```
   - 若清單為空，回傳空陣列 `{ "todos": [] }`

4. **POST `/todos`**
   - 接收 JSON 請求體 `{ "title": "..." }`
   - `id` 自動遞增（從 1 開始），`done` 預設為 `false`
   - 新增成功後以 **HTTP 201** 回傳新建立的物件：
     ```json
     { "id": 2, "title": "讀技術書", "done": false }
     ```
   - 若 `title` 未提供或為空字串，回傳 **HTTP 400**：
     ```json
     { "error": "title 為必填欄位" }
     ```

5. **GET `/todos/:id`**
   - 依 `id` 取得單筆待辦事項
   - 找不到時回傳 **HTTP 404**：
     ```json
     { "error": "找不到 id 為 3 的待辦事項" }
     ```

---

## 提示

- 使用 `express.json()` middleware 才能讀取 `req.body`：`app.use(express.json())`
- 自動遞增 id 可用陣列長度 + 1，或維護一個獨立的計數器變數
- 路由參數 `req.params.id` 取得的是**字串**，與陣列中的數字 id 比較時記得轉型：`Number(req.params.id)`
- 用 `res.status(201).json(...)` 可以同時設定狀態碼與回傳 JSON

---

## 進階挑戰（選做）

- **PATCH `/todos/:id`**：接收 `{ "done": true }` 將指定待辦事項標為完成，並回傳更新後的物件
- **DELETE `/todos/:id`**：刪除指定待辦事項，成功回傳 HTTP 204（No Content）
- 試著用 `curl` 或 Postman 測試所有路由，並記錄每個請求與回應
