# 第 1 週今日 Express 練習題：建立你的第一個 Express 伺服器

## 題目說明

你將從零開始建立一個 Express HTTP 伺服器，設定基本路由，並回傳 JSON 格式的回應。這是學習後端 API 開發的第一步：讓伺服器能夠接收 HTTP 請求並回應資料。

---

## 技術要求

- **概念：** Express 安裝、`app.get()`、`app.listen()`、JSON 回應、路由參數
- **需要的套件：** `express`

---

## 具體任務

1. 執行 `npm init -y`，然後安裝 Express：`npm install express`。

2. 建立 `app.js`，設定以下四個路由：

   | 方法 | 路徑 | 回應 |
   |------|------|------|
   | GET | `/` | `{ "message": "歡迎來到我的 API" }` |
   | GET | `/health` | `{ "status": "ok", "timestamp": <現在時間 ISO 字串> }` |
   | GET | `/users` | 回傳包含至少 3 筆假使用者資料的陣列 |
   | GET | `/users/:id` | 依 id 找到對應使用者；找不到則回傳 404 與錯誤訊息 |

3. 伺服器監聽 port `3000`，啟動時 `console.log` 提示訊息。

4. 用 `node app.js` 啟動伺服器，並用瀏覽器或 `curl` 測試所有路由。

---

## 範例

```js
// app.js
const express = require("express");
const app = express();
const PORT = 3000;

const users = [
  { id: 1, name: "Alice", email: "alice@example.com" },
  { id: 2, name: "Bob",   email: "bob@example.com" },
  { id: 3, name: "Carol", email: "carol@example.com" },
];

app.get("/", (req, res) => {
  res.json({ message: "歡迎來到我的 API" });
});

app.get("/health", (req, res) => {
  res.json({ status: "ok", timestamp: new Date().toISOString() });
});

app.get("/users", (req, res) => {
  res.json(users);
});

app.get("/users/:id", (req, res) => {
  const user = users.find((u) => u.id === Number(req.params.id));
  if (!user) {
    return res.status(404).json({ error: "找不到使用者" });
  }
  res.json(user);
});

app.listen(PORT, () => {
  console.log(`伺服器已啟動：http://localhost:${PORT}`);
});
```

**測試指令：**
```bash
curl http://localhost:3000/
curl http://localhost:3000/health
curl http://localhost:3000/users
curl http://localhost:3000/users/2
curl http://localhost:3000/users/99   # 應回傳 404
```

---

## 提示

- `res.json()` 會自動設定 `Content-Type: application/json`，不需要手動設定。
- `req.params.id` 取出的值是**字串**，要用 `Number()` 轉換才能與數字 id 比對。
- `res.status(404).json(...)` 可以鏈式呼叫，同時設定狀態碼與回應內容。
- 每次修改 `app.js` 都需要重啟伺服器（之後學 `nodemon` 可以自動重啟）。

---

## 學習重點（跟後端 API 的關聯）

這次練習對應真實 API 的骨架：

- `app.get('/')` → 健康檢查端點，讓 DevOps 確認服務存活
- `app.get('/users')` → 列表查詢端點（未來接資料庫）
- `app.get('/users/:id')` → 單筆查詢端點 + 404 錯誤處理

之後每週會逐步加上：POST（新增）、PUT/PATCH（修改）、DELETE（刪除）、Middleware、資料庫整合。

---

## 進階挑戰（選做）

1. 新增 `GET /users?name=Alice` 支援用查詢字串（`req.query.name`）篩選使用者。
2. 為不存在的路由加上 catch-all 路由，回傳 `404 { "error": "找不到此路由" }`。
3. 安裝 `nodemon`（`npm install -D nodemon`）並在 `package.json` 加入 `"dev": "nodemon app.js"` 腳本，讓伺服器在檔案變更時自動重啟。
