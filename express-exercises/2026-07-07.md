# 今日練習題：PostgreSQL 書籍管理系統 CRUD

## 題目說明
建立一個簡單的書籍管理系統，透過 Node.js 搭配 `pg` 套件連接 PostgreSQL 資料庫，實作基本的 CRUD 操作（新增、查詢、修改、刪除）。你需要設計一張 `books` 資料表，並撰寫對應的 SQL 語句與 JavaScript 函式，完成對書籍資料的完整管理功能。練習過程中請注意 SQL 注入防護，使用參數化查詢（Parameterized Query）來傳遞資料。

## 技術要求
- **Express.js**：建立 RESTful API 路由，搭配 JSON 資料格式回應
- **PostgreSQL**：建立資料表、執行 INSERT / SELECT / UPDATE / DELETE 語句，使用參數化查詢

## 具體任務

1. **建立資料表**：在 PostgreSQL 建立 `books` 資料表，欄位包含：
   - `id`（SERIAL PRIMARY KEY）
   - `title`（VARCHAR，書名）
   - `author`（VARCHAR，作者）
   - `price`（NUMERIC，價格）
   - `created_at`（TIMESTAMP，預設為當下時間）

2. **新增書籍（Create）**：實作一個函式，接收 `title`、`author`、`price`，將新書籍插入資料庫，並回傳新增後的完整資料。

3. **查詢所有書籍（Read）**：實作查詢所有書籍的函式，依照 `created_at` 降冪排序回傳清單。

4. **查詢單本書籍（Read by ID）**：透過 `id` 查詢特定書籍，若不存在則回傳適當的錯誤訊息。

5. **修改書籍（Update）**：透過 `id` 更新書籍的 `title`、`author`、`price`，並回傳更新後的資料。

6. **刪除書籍（Delete）**：透過 `id` 刪除書籍，若該 `id` 不存在則回傳錯誤提示。

## 提示
- 使用 `pg` 套件（`npm install pg`）建立連線池（`Pool`）而非單一連線（`Client`），以提升效能
- 參數化查詢寫法範例：`client.query('SELECT * FROM books WHERE id = $1', [id])`，`$1` 為佔位符，避免 SQL 注入
- 在 `created_at` 欄位可使用 `DEFAULT NOW()` 讓資料庫自動填入時間，不需在 INSERT 語句中手動傳入

## 進階挑戰（選做）
新增「搜尋功能」：實作一個查詢函式，接收關鍵字字串，使用 `ILIKE` 語句搜尋 `title` 或 `author` 欄位中包含該關鍵字的書籍，並在 Express 路由中以 `GET /books?search=關鍵字` 的形式提供此功能。
